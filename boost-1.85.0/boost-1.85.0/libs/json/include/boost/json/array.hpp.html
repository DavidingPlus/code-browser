30/** A dynamically sized array of JSON values
31
32 This is the type used to represent a JSON array as
33 a modifiable container. The interface and performance
34 characteristics are modeled after `std::vector<value>`.
35\n
36 Elements are stored contiguously, which means that
37 they can be accessed not only through iterators, but
38 also using offsets to regular pointers to elements. A
39 pointer to an element of an @ref array may be passed to
40 any function that expects a pointer to @ref value.
41\n
42 The storage of the array is handled automatically, being
43 expanded and contracted as needed. Arrays usually occupy
44 more space than array language constructs, because more
45 memory is allocated to handle future growth. This way an
46 array does not need to reallocate each time an element
47 is inserted, but only when the additional memory is used
48 up. The total amount of allocated memory can be queried
49 using the @ref capacity function. Extra memory can be
50 relinquished by calling @ref shrink_to_fit.
51 \n
52
53 Reallocations are usually costly operations in terms of
54 performance. The @ref reserve function can be used to
55 eliminate reallocations if the number of elements is
56 known beforehand.
57\n
58 The complexity (efficiency) of common operations on
59 arrays is as follows:
60
61 @li Random access - constant *O(1)*.
62 @li Insertion or removal of elements at the
63 end - amortized constant *O(1)*.
64 @li Insertion or removal of elements - linear in
65 the distance to the end of the array *O(n)*.
66
67 @par Allocators
68
69 All elements stored in the container, and their
70 children if any, will use the same memory resource
71 that was used to construct the container.
72
73 @par Thread Safety
74
75 Non-const member functions may not be called
76 concurrently with any other member functions.
77
78 @par Satisfies
79 <a href="https://en.cppreference.com/w/cpp/named_req/ContiguousContainer"><em>ContiguousContainer</em></a>,
80 <a href="https://en.cppreference.com/w/cpp/named_req/ReversibleContainer"><em>ReversibleContainer</em></a>, and
81 <a href="https://en.cppreference.com/w/cpp/named_req/SequenceContainer"><em>SequenceContainer</em></a>.
82*/

418 /** Move constructor.
419
420 The array is constructed with the contents of
421 `other` by move semantics, using the specified
422 memory resource:
423
424 @li If `*other.storage() == *sp`, ownership of
425 the underlying memory is transferred in constant
426 time, with no possibility of exceptions.
427 After construction, the moved-from array behaves
428 as if newly constructed with its current storage
429 pointer.
430
431 @li If `*other.storage() != *sp`, an
432 element-wise copy is performed, which may throw.
433 In this case, the moved-from array is not
434 changed.
435
436 @par Complexity
437 At most, linear in `other.size()`.
438
439 @par Exception Safety
440 Strong guarantee.

442

445 @param sp A pointer to the `boost::container::pmr::memory_resource`
