33/** A dynamically sized associative container of JSON key/value pairs.
34
35 This is an associative container whose elements
36 are key/value pairs with unique keys.
37\n
38 The elements are stored contiguously; iterators are
39 ordinary pointers, allowing random access pointer
40 arithmetic for retrieving elements.
41 In addition, the container maintains an internal
42 index to speed up find operations, reducing the
43 average complexity for most lookups and insertions.
44\n
45 Reallocations are usually costly operations in terms of
46 performance, as elements are copied and the internal
47 index must be rebuilt. The @ref reserve function can
48 be used to eliminate reallocations if the number of
49 elements is known beforehand.
50
51 @par Allocators
52
53 All elements stored in the container, and their
54 children if any, will use the same memory resource that
55 was used to construct the container.
56
57 @par Thread Safety
58
59 Non-const member functions may not be called
60 concurrently with any other member functions.
61
62 @par Satisfies
63 <a href="https://en.cppreference.com/w/cpp/named_req/ContiguousContainer"><em>ContiguousContainer</em></a>,
64 <a href="https://en.cppreference.com/w/cpp/named_req/ReversibleContainer"><em>ReversibleContainer</em></a>, and
65 <a href="https://en.cppreference.com/w/cpp/named_req/SequenceContainer"><em>SequenceContainer</em></a>.
66*/

325 /** Move constructor.
326
327 The object is constructed with the contents of
328 `other` by move semantics, using the specified
329 memory resource:
330
331 @li If `*other.storage() == *sp`, ownership of
332 the underlying memory is transferred in constant
333 time, with no possibility of exceptions.
334 After construction, the moved-from object behaves
335 as if newly constructed with its current storage
336 pointer.
337
338 @li If `*other.storage() != *sp`, an
339 element-wise copy is performed, which may throw.

342
343 @par Complexity
344 Constant or linear in `other.size()`.
345
346 @par Exception Safety

349

928 /** Insert elements.
929
930 Inserts `p` if `this->contains(value_type(p).key())` is `false`.
931 @ref value_type must be constructible from `p`.
932
933 If the insertion occurs and results in a rehashing
934 of the container, all iterators and references are invalidated.
935 Otherwise, they are not affected.
936 Rehashing occurs only if the new number of elements
937 is greater than @ref capacity().
938
939 @par Constraints
940 @code
941 std::is_constructible_v<value_type, P>
942 @endcode
943
944 @par Complexity
945 Average case amortized constant,
946 worst case linear in @ref size().
947
948 @par Exception Safety
949 Strong guarantee.
950 Calls to `memory_resource::allocate` may throw.
951
952 @param p The value to insert.
953
954 @throw `boost::system::system_error` key is too long.
955 @throw `boost::system::system_error` @ref size() >= max_size().
956
957 @return A pair where `first` is an iterator
958 to the existing or inserted element, and `second`
959 is `true` if the insertion took place or `false` otherwise.
960 */
961 template<class P
962#ifndef BOOST_JSON_DOCS
963 ,class = typename std::enable_if<
964 std::is_constructible<key_value_pair,
965 P, storage_ptr>::value>::type
966#endif
967 >
968 std::pair<iterator, bool>
969 insert(P&& p);
970
971 /** Insert elements.
972
973 The elements in the range `[first, last)` are inserted one at a time,
974 in order. Any element with key that is a duplicate of a key already
975 present in container will be skipped. This also means, that if there
976 are two keys within the range that are equal to each other, only the
977 first will be inserted.
978
979 Insertion may result in rehashing of the container. In that case all
