24/** A stack of @ref value elements, for building a document.
25
26 This stack of @ref value allows iterative construction of a JSON document
27 in memory. The implementation uses temporary internal storage to buffer
28 elements so that arrays, objects, and strings in the document are
29 constructed using a single memory allocation. This improves performance
30 and makes efficient use of the `boost::container::pmr::memory_resource`
31 used to create the resulting @ref value.
32
33 Temporary storage used by the implementation initially comes from an
34 optional memory buffer owned by the caller. If that storage is exhausted,
35 then memory is obtained dynamically from the
36 `boost::container::pmr::memory_resource` provided on construction.
37
38 @par Usage
39
40 Construct the stack with an optional initial temporary buffer, and a
41 @ref storage_ptr to use for more storage when the initial buffer is
42 exhausted. Then to build a @ref value, first call @ref reset and
43 optionally specify the `boost::container::pmr::memory_resource` which will
44 be used for the value. Then push elements onto the stack by calling the
45 corresponding functions. After the document has been fully created, call
46 @ref release to acquire ownership of the top-level @ref value.
47
48 @par Performance
49
50 The initial buffer and any dynamically allocated
51 temporary buffers are retained until the stack
52 is destroyed. This improves performance when using
53 a single stack instance to produce multiple
54 values.
55
56 @par Example
57
58 The following code constructs a @ref value which
59 when serialized produces a JSON object with three
60 elements. It uses a local buffer for the temporary
61 storage, and a separate local buffer for the storage
62 of the resulting value. No memory is dynamically
63 allocated; this shows how to construct a value
64 without using the heap.
65
66 @code
67
68 // This example builds a json::value without any dynamic memory allocations:
69
70 // Construct the value stack using a local buffer
71 unsigned char temp[4096];
72 value_stack st( storage_ptr(), temp, sizeof(temp) );
73
74 // Create a static resource with a local initial buffer
75 unsigned char buf[4096];
76 static_resource mr( buf, sizeof(buf) );
77
78 // All values on the stack will use `mr`
79 st.reset(&mr);
80
81 // Push the key/value pair "a":1.
82 st.push_key("a");
83 st.push_int64(1);
84
85 // Push "b":null
86 st.push_key("b");
87 st.push_null();
88
89 // Push "c":"hello"
90 st.push_key("c");
91 st.push_string("hello");
92
93 // Pop the three key/value pairs and push an object with those three values.
94 st.push_object(3);
95
96 // Pop the object from the stack and take ownership.
97 value jv = st.release();
98
99 assert( serialize(jv) == "{\"a\":1,\"b\":null,\"c\":\"hello\"}" );
100
101 // At this point we could re-use the stack by calling reset
102
103 @endcode
104
105 @par Thread Safety
106
107 Distinct instances may be accessed concurrently.
108 Non-const member functions of a shared instance
109 may not be called concurrently with any other
110 member functions of that instance.
111*/

163 /** Destructor.
164
165 All dynamically allocated memory and

168 @par Complexity

170
171 @par Exception Safety
172 No-throw guarantee.

175 ~value_stack();

177 /** Constructor.
178
179 Constructs an empty stack. Before any
180 @ref value can be built, the function
181 @ref reset must be called.
182
183 The `sp` parameter is only used to allocate
184 intermediate storage; it will not be used
185 for the @ref value returned by @ref release.
186
187 @param sp A pointer to the `boost::container::pmr::memory_resource` to
188 use for intermediate storage allocations. If this argument is omitted,
189 the default memory resource is used.
190
191 @param temp_buffer A pointer to a caller-owned
192 buffer which will be used to store temporary
193 data used while building the value. If this
194 pointer is null, the builder will use the
195 storage pointer to allocate temporary data.
196
197 @param temp_size The number of valid bytes of
198 storage pointed to by `temp_buffer`.
199 */
200 BOOST_JSON_DECL
201 value_stack(
202 storage_ptr sp = {},
203 unsigned char* temp_buffer = nullptr,
204 std::size_t temp_size = 0) noexcept;

206 /** Prepare to build a new document.
207
208 This function must be called before constructing
209 a new top-level @ref value. Any previously existing
210 partial or complete elements are destroyed, but
211 internal dynamically allocated memory is preserved
212 which may be reused to build new values.
213
214 @par Exception Safety
215
216 No-throw guarantee.
217
218 @param sp A pointer to the `boost::container::pmr::memory_resource` to
219 use for top-level @ref value and all child values. The stack will
220 acquire shared ownership of the memory resource until @ref release or
221 @ref reset is called, or when the stack is destroyed.
222 */
223 BOOST_JSON_DECL
224 void
225 reset(storage_ptr sp = {}) noexcept;
226
227 /** Return the top-level @ref value.
228
229 This function transfers ownership of the
230 constructed top-level value to the caller.
231 The behavior is undefined if there is not
232 a single, top-level element.
233
234 @par Exception Safety
235
236 No-throw guarantee.
237
238 @return A __value__ holding the result.
239 Ownership of this value is transferred
240 to the caller. Ownership of the memory
241 resource used in the last call to @ref reset
242 is released.
243 */
244 BOOST_JSON_DECL
245 value
246 release() noexcept;

250 /** Push an array formed by popping `n` values from the stack.
251
252 This function pushes an @ref array value
253 onto the stack. The array is formed by first
254 popping the top `n` values from the stack.
255 If the stack contains fewer than `n` values,
256 or if any of the top `n` values on the stack
257 is a key, the behavior is undefined.
258
259 @par Example
260
261 The following statements produce an array
262 with the contents 1, 2, 3:
263
264 @code
265
266 value_stack st;
267
268 // reset must be called first or else the behavior is undefined
269 st.reset();
270
271 // Place three values on the stack
272 st.push_int64( 1 );
273 st.push_int64( 2 );
274 st.push_int64( 3 );
275
276 // Remove the 3 values, and push an array with those 3 elements on the stack
277 st.push_array( 3 );
278
279 // Pop the object from the stack and take ownership.
280 value jv = st.release();
281
282 assert( serialize(jv) == "[1,2,3]" );
283
284 // At this point, reset must be called again to use the stack
285
286 @endcode
287
288 @param n The number of values to pop from the
289 top of the stack to form the array.
290 */
291 BOOST_JSON_DECL
292 void
293 push_array(std::size_t n);
294
295 /** Push an object formed by popping `n` key/value pairs from the stack.
296
297 This function pushes an @ref object value
298 onto the stack. The object is formed by first
299 popping the top `n` key/value pairs from the
300 stack. If the stack contains fewer than `n`
301 key/value pairs, or if any of the top `n` key/value
302 pairs on the stack does not consist of exactly one
303 key followed by one value, the behavior is undefined.
304
305 @note
306
307 A key/value pair is formed by pushing a key, and then
308 pushing a value.
309
310 @par Example
311
312 The following code creates an object on the stack
313 with a single element, where key is "x" and value
314 is true:
315
316 @code
317
318 value_stack st;
319
320 // reset must be called first or else the behavior is undefined
321 st.reset();
322
323 // Place a key/value pair onto the stack
324 st.push_key( "x" );
325 st.push_bool( true );
326
327 // Replace the key/value pair with an object containing a single element
328 st.push_object( 1 );
329
330 // Pop the object from the stack and take ownership.
331 value jv = st.release();
332
333 assert( serialize(jv) == "{\"x\",true}" );
334
335 // At this point, reset must be called again to use the stack
336
337 @endcode
338
339 @par Duplicate Keys
340
341 If there are object elements with duplicate keys;
342 that is, if multiple elements in an object have
343 keys that compare equal, only the last equivalent
344 element will be inserted.
345
346 @param n The number of key/value pairs to pop from the
347 top of the stack to form the array.
348 */
349 BOOST_JSON_DECL
350 void
351 push_object(std::size_t n);

353 /** Push part of a key or string onto the stack.
354
355 This function pushes the characters in `s` onto
356 the stack, appending to any existing characters
357 or creating new characters as needed. Once a
358 string part is placed onto the stack, the only
359 valid stack operations are:
360
361 @li @ref push_chars to append additional
362 characters to the key or string being built,
363
364 @li @ref push_key or @ref push_string to
365 finish building the key or string and place
366 the value onto the stack.
367
368 @par Exception Safety
369
370 Basic guarantee.
371 Calls to `memory_resource::allocate` may throw.
372
373 @param s The characters to append. This may be empty.
374 */
375 BOOST_JSON_DECL
376 void
377 push_chars(
378 string_view s);
