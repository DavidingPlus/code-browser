63/*!
64 * \brief A reliable interprocess message queue
65 *
66 * The queue implements a reliable one-way channel of passing messages from one or multiple writers to a single reader.
67 * The format of the messages is user-defined and must be consistent across all writers and the reader. The queue does
68 * not enforce any specific format of the messages, other than they should be supplied as a contiguous array of bytes.
69 *
70 * The queue internally uses a process-shared storage identified by an \c object_name (the queue name). Refer to \c object_name
71 * documentation for details on restrictions imposed on object names.
72 *
73 * The queue storage is organized as a fixed number of blocks of a fixed size. The block size must be an integer power of 2 and
74 * is expressed in bytes. Each written message, together with some metadata added by the queue, consumes an integer number
75 * of blocks. Each read message received by the reader releases the blocks allocated for that message. As such the maximum size
76 * of a message is slightly less than block size times capacity of the queue. For efficiency, it is recommended to choose
77 * block size large enough to accommodate most of the messages to be passed through the queue.
78 *
79 * The queue is considered empty when no messages are enqueued (all blocks are free). The queue is considered full at the point
80 * of enqueueing a message when there is not enough free blocks to accommodate the message.
81 *
82 * The queue is reliable in that it will not drop successfully sent messages that are not received by the reader, other than the
83 * case when a non-empty queue is destroyed by the last user. If a message cannot be enqueued by the writer because the queue is
84 * full, the queue can either block the writer or return an error or throw an exception, depending on the policy specified at
85 * the queue creation. The policy is object local, i.e. different writers and the reader can have different overflow policies.
86 *
87 * If the queue is empty and the reader attempts to dequeue a message, it will block until a message is enqueued by a writer.
88 *
89 * A blocked reader or writer can be unblocked by calling \c stop_local. After this method is called, all threads blocked on
90 * this particular object are released and return \c operation_result::aborted. The other instances of the queue (in the current
91 * or other processes) are unaffected. In order to restore the normal functioning of the queue instance after the \c stop_local
92 * call the user has to invoke \c reset_local.
93 *
94 * The queue does not guarantee any particular order of received messages from different writer threads. Messages sent by a
95 * particular writer thread will be received in the order of sending.
96 *
97 * Methods of this class are not thread-safe, unless otherwise specified.
98 */

102 //! Result codes for various operations on the queue

104 {

434 /*!
435 * The method wakes up all threads that are blocked in calls to <tt>send()</tt> or
436 * <tt>receive()</tt>. Those calls would then return <tt>operation_result::aborted</tt>.
437 * Note that, the method does not block until the woken-up threads have actually
438 * returned from <tt>send()</tt> or <tt>receive()</tt>. Other means is needed to ensure

447 * <tt>try_receive()</tt>, and <tt>clear()</tt> are allowed.
448 *
449 * \pre <tt>is_open() == true</tt>

479 /*!
480 * The method sends a message to the associated message queue. When the object is in
481 * running state and the queue has no free space for the message, the method either blocks
482 * or throws an exception, depending on the overflow policy that was specified on the queue
483 * opening/creation. If blocking policy is in effect, the blocking can be interrupted by
484 * calling <tt>stop_local()</tt>, in which case the method returns \c operation_result::aborted.
485 * When the object is already in the stopped state, the method does not block but returns
486 * immediately with return value \c operation_result::aborted.
487 *
488 * It is possible to send an empty message by passing \c 0 to the parameter \c message_size.

622 * Concurrent calls to <tt>send()</tt>, <tt>try_send()</tt>, <tt>receive()</tt>,
623 * <tt>try_receive()</tt>, <tt>stop_local()</tt>, and <tt>clear()</tt> are allowed.
624 *
625 * \pre <tt>is_open() == true</tt>
626 *
627 * \param buffer The memory buffer to store the received message in.
628 * \param buffer_size The size of the buffer, in bytes.
629 * \param message_size Receives the size of the received message, in bytes.
630 *
631 * \return \c true if a message is successfully received, and \c false otherwise (e.g.,
632 * when the queue is empty).
633 */
634 bool try_receive(void* buffer, size_type buffer_size, size_type& message_size)
635 {
636 fixed_buffer_state state = { .data: static_cast< uint8_t* >(buffer), .size: buffer_size };
637 bool result = do_try_receive(handler: &reliable_message_queue::fixed_buffer_receive_handler, state: &state);
638 message_size = buffer_size - state.size;
639 return result;
640 }

695 /*!
696 * The method frees system-wide resources, associated with the interprocess queue with the supplied name.
697 * The queue referred to by the specified name must not be opened in any process at the point of this call.
698 * After this call succeeds a new queue with the specified name can be created.
699 *
700 * This call can be useful to recover from an earlier process misbehavior (e.g. a crash without properly
701 * closing the message queue). In this case resources allocated for the interprocess queue may remain
702 * allocated after the last process closed the queue, which in turn may prevent creating a new queue with
703 * the same name. By calling this method before creating a queue the application can attempt to ensure
704 * it starts with a clean slate.
705 *
706 * On some platforms resources associated with the queue are automatically reclaimed by the operating system
707 * when the last process using those resources terminates (even if it terminates abnormally). On these
708 * platforms this call may be a no-op. However, portable code should still call this method at appropriate
709 * places to ensure compatibility with other platforms and future library versions, which may change implementation
710 * of the queue.
711 *
712 * \param name Name of the message queue to be removed.
713 */
