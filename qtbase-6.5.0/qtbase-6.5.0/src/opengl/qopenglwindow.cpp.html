19/*!
20 \class QOpenGLWindow
21 \inmodule QtOpenGL
22 \since 5.4
23 \brief The QOpenGLWindow class is a convenience subclass of QWindow to perform OpenGL painting.
24
25 QOpenGLWindow is an enhanced QWindow that allows easily creating windows that
26 perform OpenGL rendering using an API that is compatible with QOpenGLWidget
27 Unlike QOpenGLWidget, QOpenGLWindow has no dependency on the widgets module
28 and offers better performance.
29
30 A typical application will subclass QOpenGLWindow and reimplement the following
31 virtual functions:
32
33 \list
34
35 \li initializeGL() to perform OpenGL resource initialization
36
37 \li resizeGL() to set up the transformation matrices and other window size dependent resources
38
39 \li paintGL() to issue OpenGL commands or draw using QPainter
40
41 \endlist
42
43 To schedule a repaint, call the update() function. Note that this will not
44 immediately result in a call to paintGL(). Calling update() multiple times in
45 a row will not change the behavior in any way.
46
47 This is a slot so it can be connected to a \l QTimer::timeout() signal to
48 perform animation. Note however that in the modern OpenGL world it is a much
49 better choice to rely on synchronization to the vertical refresh rate of the
50 display. See \l{QSurfaceFormat::setSwapInterval()}{setSwapInterval()} on a
51 description of the swap interval. With a swap interval of \c 1, which is the
52 case on most systems by default, the
53 \l{QOpenGLContext::swapBuffers()}{swapBuffers()} call, that is executed
54 internally by QOpenGLWindow after each repaint, will block and wait for
55 vsync. This means that whenever the swap is done, an update can be scheduled
56 again by calling update(), without relying on timers.
57
58 To request a specific configuration for the context, use setFormat()
59 like for any other QWindow. This allows, among others, requesting a
60 given OpenGL version and profile, or enabling depth and stencil
61 buffers.
62
63 Unlike QWindow, QOpenGLWindow allows opening a painter on itself and perform
64 QPainter-based drawing.
65
66 QOpenGLWindow supports multiple update behaviors. The default,
67 \c NoPartialUpdate is equivalent to a regular, OpenGL-based QWindow. In
68 contrast, \c PartialUpdateBlit and \c PartialUpdateBlend are
69 more in line with QOpenGLWidget's way of working, where there is always an
70 extra, dedicated framebuffer object present. These modes allow, by
71 sacrificing some performance, redrawing only a smaller area on each paint and
72 having the rest of the content preserved from of the previous frame. This is
73 useful for applications than render incrementally using QPainter, because
74 this way they do not have to redraw the entire window content on each
75 paintGL() call.
76
77 Similarly to QOpenGLWidget, QOpenGLWindow supports the Qt::AA_ShareOpenGLContexts
78 attribute. When enabled, the OpenGL contexts of all QOpenGLWindow instances will share
79 with each other. This allows accessing each other's shareable OpenGL resources.
80
81 For more information on graphics in Qt, see \l {Graphics}.
82 */
83
84/*!
85 \enum QOpenGLWindow::UpdateBehavior
86
87 This enum describes the update strategy of the QOpenGLWindow.
88
89 \value NoPartialUpdate Indicates that the entire window surface will
90 redrawn on each update and so no additional framebuffers are needed.
91 This is the setting used in most cases and is equivalent to how drawing
92 directly via QWindow would function.
93
94 \value PartialUpdateBlit Indicates that the drawing performed in paintGL()
95 does not cover the entire window. In this case an extra framebuffer object
96 is created under the hood, and rendering performed in paintGL() will target
97 this framebuffer. This framebuffer is then blitted onto the window surface's
98 default framebuffer after each paint. This allows having QPainter-based drawing
99 code in paintGL() which only repaints a smaller area at a time, because, unlike
100 NoPartialUpdate, the previous content is preserved.
101
102 \value PartialUpdateBlend Similar to PartialUpdateBlit, but instead of using
103 framebuffer blits, the contents of the extra framebuffer is rendered by
104 drawing a textured quad with blending enabled. This, unlike PartialUpdateBlit,
105 allows alpha blended content and works even when the glBlitFramebuffer is
106 not available. Performance-wise this setting is likely to be somewhat slower
107 than PartialUpdateBlit.
108 */
109
110/*!
111 \fn void QOpenGLWindow::frameSwapped()
112
113 This signal is emitted after the potentially blocking
114 \l{QOpenGLContext::swapBuffers()}{buffer swap} has been done. Applications
115 that wish to continuously repaint synchronized to the vertical refresh,
116 should issue an update() upon this signal. This allows for a much smoother
117 experience compared to the traditional usage of timers.
118*/

172QOpenGLWindowPrivate::~QOpenGLWindowPrivate()
173{
174 Q_Q(QOpenGLWindow);
175 if (q->isValid()) {
176 q->makeCurrent(); // this works even when the platformwindow is destroyed
177 paintDevice.reset(other: nullptr);
178 fbo.reset(other: nullptr);
179 blitter.destroy();
180 q->doneCurrent();
181 }

184void QOpenGLWindowPrivate::initialize()
185{
186 Q_Q(QOpenGLWindow);
187
188 if (context)
189 return;
190
191 if (!q->handle())
192 qWarning(msg: "Attempted to initialize QOpenGLWindow without a platform window");
193
194 context.reset(other: new QOpenGLContext);
195 context->setShareContext(shareContext);
196 context->setFormat(q->requestedFormat());
197 if (!context->create())
198 qWarning(msg: "QOpenGLWindow::beginPaint: Failed to create context");
199 if (!context->makeCurrent(surface: q))
200 qWarning(msg: "QOpenGLWindow::beginPaint: Failed to make context current");
201
202 paintDevice.reset(other: new QOpenGLWindowPaintDevice(q));
203 if (updateBehavior == QOpenGLWindow::PartialUpdateBlit)
204 hasFboBlit = QOpenGLFramebufferObject::hasOpenGLFramebufferBlit();
205
206 q->initializeGL();
207}

209void QOpenGLWindowPrivate::beginPaint(const QRegion &region)
210{
211 Q_UNUSED(region);
212 Q_Q(QOpenGLWindow);
213
214 initialize();
215 context->makeCurrent(surface: q);
216
217 const int deviceWidth = q->width() * q->devicePixelRatio();
218 const int deviceHeight = q->height() * q->devicePixelRatio();
219 const QSize deviceSize(deviceWidth, deviceHeight);
220 if (updateBehavior > QOpenGLWindow::NoPartialUpdate) {
221 if (!fbo || fbo->size() != deviceSize) {
222 QOpenGLFramebufferObjectFormat fboFormat;
223 fboFormat.setAttachment(QOpenGLFramebufferObject::CombinedDepthStencil);
224 const int samples = q->requestedFormat().samples();
225 if (samples > 0) {
226 if (updateBehavior != QOpenGLWindow::PartialUpdateBlend)
227 fboFormat.setSamples(samples);
228 else
229 qWarning(msg: "QOpenGLWindow: PartialUpdateBlend does not support multisampling");
230 }
231 fbo.reset(other: new QOpenGLFramebufferObject(deviceSize, fboFormat));
232 markWindowAsDirty();
233 }
234 } else {
235 markWindowAsDirty();
236 }
237
238 paintDevice->setSize(QSize(deviceWidth, deviceHeight));
239 paintDevice->setDevicePixelRatio(q->devicePixelRatio());
240 context->functions()->glViewport(x: 0, y: 0, width: deviceWidth, height: deviceHeight);
241
242 context->functions()->glBindFramebuffer(GL_FRAMEBUFFER, framebuffer: context->defaultFramebufferObject());
243
244 q->paintUnderGL();
245
246 if (updateBehavior > QOpenGLWindow::NoPartialUpdate)
247 fbo->bind();
248}
249
250void QOpenGLWindowPrivate::endPaint()
251{
252 Q_Q(QOpenGLWindow);
253
254 if (updateBehavior > QOpenGLWindow::NoPartialUpdate)
255 fbo->release();
256
257 context->functions()->glBindFramebuffer(GL_FRAMEBUFFER, framebuffer: context->defaultFramebufferObject());
258
259 if (updateBehavior == QOpenGLWindow::PartialUpdateBlit && hasFboBlit) {
260 const int deviceWidth = q->width() * q->devicePixelRatio();
261 const int deviceHeight = q->height() * q->devicePixelRatio();
262 QOpenGLExtensions extensions(context.data());
263 extensions.glBindFramebuffer(GL_READ_FRAMEBUFFER, framebuffer: fbo->handle());
264 extensions.glBindFramebuffer(GL_DRAW_FRAMEBUFFER, framebuffer: context->defaultFramebufferObject());
265 extensions.glBlitFramebuffer(srcX0: 0, srcY0: 0, srcX1: deviceWidth, srcY1: deviceHeight,
266 dstX0: 0, dstY0: 0, dstX1: deviceWidth, dstY1: deviceHeight,
267 GL_COLOR_BUFFER_BIT, GL_NEAREST);
268 } else if (updateBehavior > QOpenGLWindow::NoPartialUpdate) {
269 if (updateBehavior == QOpenGLWindow::PartialUpdateBlend) {
270 context->functions()->glEnable(GL_BLEND);
271 context->functions()->glBlendFunc(GL_SRC_ALPHA, GL_ONE_MINUS_SRC_ALPHA);
272 }
273 if (!blitter.isCreated())
274 blitter.create();
275
276 QRect windowRect(QPoint(0, 0), fbo->size());
277 QMatrix4x4 target = QOpenGLTextureBlitter::targetTransform(target: windowRect, viewport: windowRect);
278 blitter.bind();
279 blitter.blit(texture: fbo->texture(), targetTransform: target, sourceOrigin: QOpenGLTextureBlitter::OriginBottomLeft);
280 blitter.release();
281
282 if (updateBehavior == QOpenGLWindow::PartialUpdateBlend)
283 context->functions()->glDisable(GL_BLEND);
284 }
285
286 q->paintOverGL();
287}

332/*!
333 Destroys the QOpenGLWindow instance, freeing its resources.
334
335 The OpenGLWindow's context is made current in the destructor, allowing for
336 safe destruction of any child object that may need to release OpenGL
337 resources belonging to the context provided by this window.
338
339 \warning if you have objects wrapping OpenGL resources (such as
340 QOpenGLBuffer, QOpenGLShaderProgram, etc.) as members of a QOpenGLWindow
341 subclass, you may need to add a call to makeCurrent() in that subclass'
342 destructor as well. Due to the rules of C++ object destruction, those objects
343 will be destroyed \e{before} calling this function (but after that the
344 destructor of the subclass has run), therefore making the OpenGL context
345 current in this function happens too late for their safe disposal.
346
347 \sa makeCurrent
348
349 \since 5.5
350*/
351QOpenGLWindow::~QOpenGLWindow()
352{
353 makeCurrent();
354}

376/*!
377 Prepares for rendering OpenGL content for this window by making the
378 corresponding context current and binding the framebuffer object, if there is
379 one, in that context context.
380
381 It is not necessary to call this function in most cases, because it is called
382 automatically before invoking paintGL(). It is provided nonetheless to support
383 advanced, multi-threaded scenarios where a thread different than the GUI or main
384 thread may want to update the surface or framebuffer contents. See QOpenGLContext
385 for more information on threading related issues.
386
387 This function is suitable for calling also when the underlying platform window
388 is already destroyed. This means that it is safe to call this function from
389 a QOpenGLWindow subclass' destructor. If there is no native window anymore,
390 an offscreen surface is used instead. This ensures that OpenGL resource
391 cleanup operations in the destructor will always work, as long as
392 this function is called first.
393
394 \sa QOpenGLContext, context(), paintGL(), doneCurrent()
395 */
396void QOpenGLWindow::makeCurrent()
397{
398 Q_D(QOpenGLWindow);
399
400 if (!isValid())
401 return;
402
403 // The platform window may be destroyed at this stage and therefore
404 // makeCurrent() may not safely be called with 'this'.
405 if (handle()) {
406 d->context->makeCurrent(surface: this);
407 } else {
408 if (!d->offscreenSurface) {
409 d->offscreenSurface.reset(other: new QOffscreenSurface(screen()));
410 d->offscreenSurface->setFormat(d->context->format());
411 d->offscreenSurface->create();
412 }
413 d->context->makeCurrent(surface: d->offscreenSurface.data());
414 }
415
416 d->bindFBO();
417}

419/*!
420 Releases the context.
421
422 It is not necessary to call this function in most cases, since the widget
423 will make sure the context is bound and released properly when invoking
424 paintGL().
425
426 \sa makeCurrent()
427 */
428void QOpenGLWindow::doneCurrent()
429{
430 Q_D(QOpenGLWindow);
431
432 if (!isValid())
433 return;
434
435 d->context->doneCurrent();
436}

456/*!
457 The framebuffer object handle used by this window.
458
459 When the update behavior is set to \c NoPartialUpdate, there is no separate
460 framebuffer object. In this case the returned value is the ID of the
461 default framebuffer.
462
463 Otherwise the value of the ID of the framebuffer object or \c 0 if not
464 yet initialized.
465 */
466GLuint QOpenGLWindow::defaultFramebufferObject() const
467{
468 Q_D(const QOpenGLWindow);
469 if (d->updateBehavior > NoPartialUpdate && d->fbo)
470 return d->fbo->handle();
471 else if (QOpenGLContext *ctx = QOpenGLContext::currentContext())
472 return ctx->defaultFramebufferObject();
473 else
474 return 0;
475}

477/*!
478 Returns a copy of the framebuffer.
479
480 \note This is a potentially expensive operation because it relies on
481 glReadPixels() to read back the pixels. This may be slow and can stall the
482 GPU pipeline.
483
484 \note When used together with update behavior \c NoPartialUpdate, the returned
485 image may not contain the desired content when called after the front and back
486 buffers have been swapped (unless preserved swap is enabled in the underlying
487 windowing system interface). In this mode the function reads from the back
488 buffer and the contents of that may not match the content on the screen (the
489 front buffer). In this case the only place where this function can safely be
490 used is paintGL() or paintOverGL().
491 */
492QImage QOpenGLWindow::grabFramebuffer()
493{
494 if (!isValid())
495 return QImage();
496
497 makeCurrent();
498
499 const bool hasAlpha = format().hasAlpha();
500 QImage img = qt_gl_read_framebuffer(size: size() * devicePixelRatio(), alpha_format: hasAlpha, include_alpha: hasAlpha);
501 img.setDevicePixelRatio(devicePixelRatio());
502 return img;
503}

567/*!
568 The virtual function is called before each invocation of paintGL().
569
570 When the update mode is set to \c NoPartialUpdate, there is no difference
571 between this function and paintGL(), performing rendering in either of them
572 leads to the same result.
573
574 The difference becomes significant when using \c PartialUpdateBlend, where an
575 extra framebuffer object is used. There, paintGL() targets this additional
576 framebuffer object, which preserves its contents, while paintUnderGL() and
577 paintOverGL() target the default framebuffer, i.e. directly the window
578 surface, the contents of which is lost after each displayed frame.
579
580 \note Avoid relying on this function when the update behavior is
581 \c PartialUpdateBlit. This mode involves blitting the extra framebuffer used by
582 paintGL() onto the default framebuffer after each invocation of paintGL(),
583 thus overwriting all drawing generated in this function.
584
585 \sa paintGL(), paintOverGL(), UpdateBehavior
586 */

588{
589}
