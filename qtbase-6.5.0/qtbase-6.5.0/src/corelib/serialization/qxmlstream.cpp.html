279/*!
280 \class QXmlStreamReader
281 \inmodule QtCore
282 \reentrant
283 \since 4.3
284
285 \brief The QXmlStreamReader class provides a fast parser for reading
286 well-formed XML via a simple streaming API.
287
288
289 \ingroup xml-tools
290
291 \ingroup qtserialization
292
293 QXmlStreamReader provides a simple streaming API to parse well-formed
294 XML. It is an alternative to first loading the complete XML into a
295 DOM tree (see \l QDomDocument). QXmlStreamReader reads data either
296 from a QIODevice (see setDevice()), or from a raw QByteArray (see addData()).
297
298 Qt provides QXmlStreamWriter for writing XML.
299
300 The basic concept of a stream reader is to report an XML document as
301 a stream of tokens, similar to SAX. The main difference between
302 QXmlStreamReader and SAX is \e how these XML tokens are reported.
303 With SAX, the application must provide handlers (callback functions)
304 that receive so-called XML \e events from the parser at the parser's
305 convenience. With QXmlStreamReader, the application code itself
306 drives the loop and pulls \e tokens from the reader, one after
307 another, as it needs them. This is done by calling readNext(), where
308 the reader reads from the input stream until it completes the next
309 token, at which point it returns the tokenType(). A set of
310 convenient functions including isStartElement() and text() can then
311 be used to examine the token to obtain information about what has
312 been read. The big advantage of this \e pulling approach is the
313 possibility to build recursive descent parsers with it, meaning you
314 can split your XML parsing code easily into different methods or
315 classes. This makes it easy to keep track of the application's own
316 state when parsing XML.
317
318 A typical loop with QXmlStreamReader looks like this:
319
320 \snippet code/src_corelib_xml_qxmlstream.cpp 0
321
322
323 QXmlStreamReader is a well-formed XML 1.0 parser that does \e not
324 include external parsed entities. As long as no error occurs, the
325 application code can thus be assured that the data provided by the
326 stream reader satisfies the W3C's criteria for well-formed XML. For
327 example, you can be certain that all tags are indeed nested and
328 closed properly, that references to internal entities have been
329 replaced with the correct replacement text, and that attributes have
330 been normalized or added according to the internal subset of the
331 DTD.
332
333 If an error occurs while parsing, atEnd() and hasError() return
334 true, and error() returns the error that occurred. The functions
335 errorString(), lineNumber(), columnNumber(), and characterOffset()
336 are for constructing an appropriate error or warning message. To
337 simplify application code, QXmlStreamReader contains a raiseError()
338 mechanism that lets you raise custom errors that trigger the same
339 error handling described.
340
341 The \l{QXmlStream Bookmarks Example} illustrates how to use the
342 recursive descent technique to read an XML bookmark file (XBEL) with
343 a stream reader.
344
345 \section1 Namespaces
346
347 QXmlStream understands and resolves XML namespaces. E.g. in case of
348 a StartElement, namespaceUri() returns the namespace the element is
349 in, and name() returns the element's \e local name. The combination
350 of namespaceUri and name uniquely identifies an element. If a
351 namespace prefix was not declared in the XML entities parsed by the
352 reader, the namespaceUri is empty.
353
354 If you parse XML data that does not utilize namespaces according to
355 the XML specification or doesn't use namespaces at all, you can use
356 the element's qualifiedName() instead. A qualified name is the
357 element's prefix() followed by colon followed by the element's local
358 name() - exactly like the element appears in the raw XML data. Since
359 the mapping namespaceUri to prefix is neither unique nor universal,
360 qualifiedName() should be avoided for namespace-compliant XML data.
361
362 In order to parse standalone documents that do use undeclared
363 namespace prefixes, you can turn off namespace processing completely
364 with the \l namespaceProcessing property.
365
366 \section1 Incremental Parsing
367
368 QXmlStreamReader is an incremental parser. It can handle the case
369 where the document can't be parsed all at once because it arrives in
370 chunks (e.g. from multiple files, or over a network connection).
371 When the reader runs out of data before the complete document has
372 been parsed, it reports a PrematureEndOfDocumentError. When more
373 data arrives, either because of a call to addData() or because more
374 data is available through the network device(), the reader recovers
375 from the PrematureEndOfDocumentError error and continues parsing the
376 new data with the next call to readNext().
377
378 For example, if your application reads data from the network using a
379 \l{QNetworkAccessManager} {network access manager}, you would issue
380 a \l{QNetworkRequest} {network request} to the manager and receive a
381 \l{QNetworkReply} {network reply} in return. Since a QNetworkReply
382 is a QIODevice, you connect its \l{QIODevice::readyRead()}
383 {readyRead()} signal to a custom slot, e.g. \c{slotReadyRead()} in
384 the code snippet shown in the discussion for QNetworkAccessManager.
385 In this slot, you read all available data with
386 \l{QIODevice::readAll()} {readAll()} and pass it to the XML
387 stream reader using addData(). Then you call your custom parsing
388 function that reads the XML events from the reader.
389
390 \section1 Performance and Memory Consumption
391
392 QXmlStreamReader is memory-conservative by design, since it doesn't
393 store the entire XML document tree in memory, but only the current
394 token at the time it is reported. In addition, QXmlStreamReader
395 avoids the many small string allocations that it normally takes to
396 map an XML document to a convenient and Qt-ish API. It does this by
397 reporting all string data as QStringView rather than real QString
398 objects. Calling \l{QStringView::toString()}{toString()} on any of
399 those objects returns an equivalent real QString object.
400*/

595/*!
596 Returns \c true if the reader has read until the end of the XML
597 document, or if an error() has occurred and reading has been
598 aborted. Otherwise, it returns \c false.
599
600 When atEnd() and hasError() return true and error() returns
601 PrematureEndOfDocumentError, it means the XML has been well-formed
602 so far, but a complete XML document has not been parsed. The next
603 chunk of XML can be added with addData(), if the XML is being read
604 from a QByteArray, or by waiting for more data to arrive if the
605 XML is being read from a QIODevice. Either way, atEnd() will
606 return false once more data is available.
607
608 \sa hasError(), error(), device(), QIODevice::atEnd()
609 */
610bool QXmlStreamReader::atEnd() const
611{
612 Q_D(const QXmlStreamReader);
613 if (d->atEnd
614 && ((d->type == QXmlStreamReader::Invalid && d->error == PrematureEndOfDocumentError)
615 || (d->type == QXmlStreamReader::EndDocument))) {
616 if (d->device)
617 return d->device->atEnd();
618 else
619 return !d->dataBuffer.size();
620 }
621 return (d->atEnd || d->type == QXmlStreamReader::Invalid);
622}

625/*!
626 Reads the next token and returns its type.
627
628 With one exception, once an error() is reported by readNext(),
629 further reading of the XML stream is not possible. Then atEnd()
630 returns \c true, hasError() returns \c true, and this function returns
631 QXmlStreamReader::Invalid.
632
633 The exception is when error() returns PrematureEndOfDocumentError.
634 This error is reported when the end of an otherwise well-formed
635 chunk of XML is reached, but the chunk doesn't represent a complete
636 XML document. In that case, parsing \e can be resumed by calling
637 addData() to add the next chunk of XML, when the stream is being
638 read from a QByteArray, or by waiting for more data to arrive when
639 the stream is being read from a device().
640
641 \sa tokenType(), tokenString()
642 */
643QXmlStreamReader::TokenType QXmlStreamReader::readNext()
644{
645 Q_D(QXmlStreamReader);
646 if (d->type != Invalid) {
647 if (!d->hasCheckedStartDocument)
648 if (!d->checkStartDocument())
649 return d->type; // synthetic StartDocument or error
650 d->parse();
651 if (d->atEnd && d->type != EndDocument && d->type != Invalid)
652 d->raiseError(error: PrematureEndOfDocumentError);
653 else if (!d->atEnd && d->type == EndDocument)
654 d->raiseWellFormedError(message: QXmlStream::tr(sourceText: "Extra content at end of document."));
655 } else if (d->error == PrematureEndOfDocumentError) {
656 // resume error
657 d->type = NoToken;
658 d->atEnd = false;
659 d->token = -1;
660 return readNext();
661 }
662 return d->type;
663}

682/*!
683 Reads until the next start element within the current element. Returns \c true
684 when a start element was reached. When the end element was reached, or when
685 an error occurred, false is returned.
686
687 The current element is the element matching the most recently parsed start
688 element of which a matching end element has not yet been reached. When the
689 parser has reached the end element, the current element becomes the parent
690 element.
691
692 This is a convenience function for when you're only concerned with parsing
693 XML elements. The \l{QXmlStream Bookmarks Example} makes extensive use of
694 this function.
695
696 \since 4.6
697 \sa readNext()
698 */
699bool QXmlStreamReader::readNextStartElement()
700{
701 while (readNext() != Invalid) {
702 if (isEndElement())
703 return false;
704 else if (isStartElement())
705 return true;
706 }
707 return false;
708}

710/*!
711 Reads until the end of the current element, skipping any child nodes.
712 This function is useful for skipping unknown elements.
713
714 The current element is the element matching the most recently parsed start
715 element of which a matching end element has not yet been reached. When the
716 parser has reached the end element, the current element becomes the parent
717 element.
718
719 \since 4.6
720 */
721void QXmlStreamReader::skipCurrentElement()
722{
723 int depth = 1;
724 while (depth && readNext() != Invalid) {
725 if (isEndElement())
726 --depth;
727 else if (isStartElement())
728 ++depth;
729 }
730}

962/*!
963 \internal
964
965 Scans characters until \a str is encountered, and validates the characters
966 as according to the Char[2] production and do the line-ending normalization.
967 If any character is invalid, false is returned, otherwise true upon success.
968
969 If \a tokenToInject is not less than zero, injectToken() is called with
970 \a tokenToInject when \a str is found.
971
972 If any error occurred, false is returned, otherwise true.
973 */
974bool QXmlStreamReaderPrivate::scanUntil(const char *str, short tokenToInject)
975{
976 const qsizetype pos = textBuffer.size();
977 const auto oldLineNumber = lineNumber;
978
979 uint c;
980 while ((c = getChar()) != StreamEOF) {
981 /* First, we do the validation & normalization. */
982 switch (c) {
983 case '\r':
984 if ((c = filterCarriageReturn()) == 0)
985 break;
986 Q_FALLTHROUGH();
987 case '\n':
988 ++lineNumber;
989 lastLineStart = characterOffset + readBufferPos;
990 Q_FALLTHROUGH();
991 case '\t':
992 textBuffer += QChar(c);
993 continue;
994 default:
995 if (c < 0x20 || (c > 0xFFFD && c < 0x10000) || c > QChar::LastValidCodePoint ) {
996 raiseWellFormedError(message: QXmlStream::tr(sourceText: "Invalid XML character."));
997 lineNumber = oldLineNumber;
998 return false;
999 }
1000 textBuffer += QChar(c);
1001 }
1002
1003
1004 /* Second, attempt to lookup str. */
1005 if (c == uint(*str)) {
1006 if (!*(str + 1)) {
1007 if (tokenToInject >= 0)
1008 injectToken(tokenToInject);
1009 return true;
1010 } else {
1011 if (scanString(str: str + 1, tokenToInject, requireSpace: false))
1012 return true;
1013 }
1014 }
1015 }
1016 putString(s: textBuffer, from: pos);
1017 textBuffer.resize(size: pos);
1018 lineNumber = oldLineNumber;
1019 return false;
1020}

1484{

1486 characterOffset += readBufferPos;
1487 readBufferPos = 0;
1488 if (readBuffer.size())
1489 readBuffer.resize(size: 0);
