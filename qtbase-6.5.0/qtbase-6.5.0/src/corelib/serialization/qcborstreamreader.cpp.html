59/*!
60 \class QCborStreamReader
61 \inmodule QtCore
62 \ingroup cbor
63 \ingroup qtserialization
64 \reentrant
65 \since 5.12
66
67 \brief The QCborStreamReader class is a simple CBOR stream decoder, operating
68 on either a QByteArray or QIODevice.
69
70 This class can be used to decode a stream of CBOR content directly from
71 either a QByteArray or a QIODevice. CBOR is the Concise Binary Object
72 Representation, a very compact form of binary data encoding that is
73 compatible with JSON. It was created by the IETF Constrained RESTful
74 Environments (CoRE) WG, which has used it in many new RFCs. It is meant to
75 be used alongside the \l{RFC 7252}{CoAP
76 protocol}.
77
78 QCborStreamReader provides a StAX-like API, similar to that of
79 \l{QXmlStreamReader}. Using it requires a bit of knowledge of CBOR encoding.
80 For a simpler API, see \l{QCborValue} and especially the decoding function
81 QCborValue::fromCbor().
82
83 Typically, one creates a QCborStreamReader by passing the source QByteArray
84 or QIODevice as a parameter to the constructor, then pop elements off the
85 stream if there were no errors in decoding. There are three kinds of CBOR
86 types:
87
88 \table
89 \header \li Kind \li Types \li Behavior
90 \row \li Fixed-width \li Integers, Tags, Simple types, Floating point
91 \li Value is pre-parsed by QCborStreamReader, so accessor functions
92 are \c const. Must call next() to advance.
93 \row \li Strings \li Byte arrays, Text strings
94 \li Length (if known) is pre-parsed, but the string itself is not.
95 The accessor functions are not const and may allocate memory.
96 Once called, the accessor functions automatically advance to
97 the next element.
98 \row \li Containers \li Arrays, Maps
99 \li Length (if known) is pre-parsed. To access the elements, you
100 must call enterContainer(), read all elements, then call
101 leaveContainer(). That function advances to the next element.
102 \endtable
103
104 So a processor function typically looks like this:
105
106 \snippet code/src_corelib_serialization_qcborstream.cpp 24
107
108 \section1 CBOR support
109
110 The following table lists the CBOR features that QCborStreamReader supports.
111
112 \table
113 \header \li Feature \li Support
114 \row \li Unsigned numbers \li Yes (full range)
115 \row \li Negative numbers \li Yes (full range)
116 \row \li Byte strings \li Yes
117 \row \li Text strings \li Yes
118 \row \li Chunked strings \li Yes
119 \row \li Tags \li Yes (arbitrary)
120 \row \li Booleans \li Yes
121 \row \li Null \li Yes
122 \row \li Undefined \li Yes
123 \row \li Arbitrary simple values \li Yes
124 \row \li Half-precision float (16-bit) \li Yes
125 \row \li Single-precision float (32-bit) \li Yes
126 \row \li Double-precision float (64-bit) \li Yes
127 \row \li Infinities and NaN floating point \li Yes
128 \row \li Determinate-length arrays and maps \li Yes
129 \row \li Indeterminate-length arrays and maps \li Yes
130 \row \li Map key types other than strings and integers \li Yes (arbitrary)
131 \endtable
132
133 \section1 Dealing with invalid or incomplete CBOR streams
134
135 QCborStreamReader is capable of detecting corrupt input on its own. The
136 library it uses has been extensively tested against invalid input of any
137 kind and is quite able to report errors. If any is detected,
138 QCborStreamReader will set lastError() to a value besides
139 QCborError::NoError, indicating which situation was detected.
140
141 Most errors detected by QCborStreamReader during normal item parsing are not
142 recoverable. The code using QCborStreamReader may opt to handle the data
143 that was properly decoded or it can opt to discard the entire data.
144
145 The only recoverable error is QCborError::EndOfFile, which indicates that
146 more data is required in order to complete the parsing. This situation is
147 useful when data is being read from an asynchronous source, such as a pipe
148 (QProcess) or a socket (QTcpSocket, QUdpSocket, QNetworkReply, etc.). When
149 more data arrives, the surrounding code needs to call either addData(), if
150 parsing from a QByteArray, or reparse(), if it is instead reading directly
151 a the QIDOevice that now has more data available (see setDevice()).
152
153 \sa QCborStreamWriter, QCborValue, QXmlStreamReader, {Cbordump Example}
154 \sa {Convert Example}, {JSON Save Game Example}
155 */
156
157/*!
158 \enum QCborStreamReader::Type
159
160 This enumeration contains all possible CBOR types as decoded by
161 QCborStreamReader. CBOR has 7 major types, plus a number of simple types
162 carrying no value, and floating point values.
163
164 \value UnsignedInteger (Major type 0) Ranges from 0 to 2\sup{64} - 1
165 (18,446,744,073,709,551,616)
166 \value NegativeInteger (Major type 1) Ranges from -1 to -2\sup{64}
167 (-18,446,744,073,709,551,616)
168 \value ByteArray (Major type 2) Arbitrary binary data.
169 \value ByteString An alias to ByteArray.
170 \value String (Major type 3) Unicode text, possibly containing NULs.
171 \value TextString An alias to String
172 \value Array (Major type 4) Array of heterogeneous items.
173 \value Map (Major type 5) Map/dictionary of heterogeneous items.
174 \value Tag (Major type 6) Numbers giving further semantic value
175 to generic CBOR items. See \l QCborTag for more information.
176 \value SimpleType (Major type 7) Types carrying no further value. Includes
177 booleans (true and false), null, undefined.
178 \value Float16 IEEE 754 half-precision floating point (\c qfloat16).
179 \value HalfFloat An alias to Float16.
180 \value Float IEEE 754 single-precision floating point (\tt float).
181 \value Double IEEE 754 double-precision floating point (\tt double).
182 \value Invalid Not a valid type, either due to parsing error or due to
183 reaching the end of an array or map.

514class QCborStreamReaderPrivate
515{
516public:
517 enum {
518 // 9 bytes is the maximum size for any integer, floating point or
519 // length in CBOR.
520 MaxCborIndividualSize = 9,
521 IdealIoBufferSize = 256
522 };
523
524 QIODevice *device;
525 QByteArray buffer;
526 QStack<CborValue> containerStack;
527
528 CborParser parser;
529 CborValue currentElement;
530 QCborError lastError = {};
531
532 QByteArray::size_type bufferStart = 0;
533 bool corrupt = false;
534
535 QCborStreamReaderPrivate(const QByteArray &data)
536 : device(nullptr), buffer(data)
537 {
538 initDecoder();
539 }
540
541 QCborStreamReaderPrivate(QIODevice *device)
542 {
543 setDevice(device);
544 }
545
546 ~QCborStreamReaderPrivate()
547 {
548 }
549
550 void setDevice(QIODevice *dev)
551 {
552 buffer.clear();
553 device = dev;
554 initDecoder();
555 }
556
557 void initDecoder()
558 {
559 containerStack.clear();
560 bufferStart = 0;
561 if (device) {
562 buffer.clear();
563 buffer.reserve(asize: IdealIoBufferSize); // sets the CapacityReserved flag
564 }
565
566 preread();
567 if (CborError err = cbor_parser_init_reader(ops: nullptr, parser: &parser, it: &currentElement, token: this))
568 handleError(err);
569 else
570 lastError = { .c: QCborError::NoError };
571 }
572
573 char *bufferPtr()
574 {
575 Q_ASSERT(buffer.isDetached());
576 return const_cast<char *>(buffer.constData()) + bufferStart;
577 }
578
579 void preread()
580 {
581 if (device && buffer.size() - bufferStart < MaxCborIndividualSize) {
582 // load more, but only if there's more to be read
583 qint64 avail = device->bytesAvailable();
584 Q_ASSERT(avail >= buffer.size());
585 if (avail == buffer.size())
586 return;
587
588 if (bufferStart)
589 device->skip(maxSize: bufferStart); // skip what we've already parsed
590
591 if (buffer.size() != IdealIoBufferSize)
592 buffer.resize(size: IdealIoBufferSize);
593
594 bufferStart = 0;
595 qint64 read = device->peek(data: bufferPtr(), maxlen: IdealIoBufferSize);
596 if (read < 0)
597 buffer.clear();
598 else if (read != IdealIoBufferSize)
599 buffer.truncate(pos: read);
600 }
601 }
602
603 void handleError(CborError err) noexcept
604 {
605 Q_ASSERT(err);
606
607 // is the error fatal?
608 if (err != CborErrorUnexpectedEOF)
609 corrupt = true;
610
611 lastError = QCborError { .c: QCborError::Code(int(err)) };
612 }

643static inline bool qt_cbor_decoder_can_read(void *token, size_t len)
644{
645 Q_ASSERT(len <= QCborStreamReaderPrivate::MaxCborIndividualSize);
646 auto self = static_cast<QCborStreamReaderPrivate *>(token);
647
648 qint64 avail = self->buffer.size() - self->bufferStart;
649 return len <= quint64(avail);
650}
651
652static void qt_cbor_decoder_advance(void *token, size_t len)
653{
654 Q_ASSERT(len <= QCborStreamReaderPrivate::MaxCborIndividualSize);
655 auto self = static_cast<QCborStreamReaderPrivate *>(token);
656 Q_ASSERT(len <= size_t(self->buffer.size() - self->bufferStart));
657
658 self->bufferStart += int(len);
659 self->preread();
660}

662static void *qt_cbor_decoder_read(void *token, void *userptr, size_t offset, size_t len)
663{
664 Q_ASSERT(len == 1 || len == 2 || len == 4 || len == 8);
665 Q_ASSERT(offset == 0 || offset == 1);
666 auto self = static_cast<const QCborStreamReaderPrivate *>(token);
667
668 // we must have pre-read the data
669 Q_ASSERT(len + offset <= size_t(self->buffer.size() - self->bufferStart));
670 return memcpy(dest: userptr, src: self->buffer.constData() + self->bufferStart + offset, n: len);
671}

673static CborError qt_cbor_decoder_transfer_string(void *token, const void **userptr, size_t offset, size_t len)
674{
675 auto self = static_cast<QCborStreamReaderPrivate *>(token);
676 Q_ASSERT(offset <= size_t(self->buffer.size()));
677 static_assert(sizeof(size_t) >= sizeof(QByteArray::size_type));
678 static_assert(sizeof(size_t) == sizeof(qsizetype));
679
680 // check that we will have enough data from the QIODevice before we advance
681 // (otherwise, we'd lose the length information)
682 qsizetype total;
683 if (len > size_t(std::numeric_limits<QByteArray::size_type>::max())
684 || add_overflow<qsizetype>(v1: offset, v2: len, r: &total))
685 return CborErrorDataTooLarge;
686
687 // our string transfer is just saving the offset to the userptr
688 *userptr = reinterpret_cast<void *>(offset);
689
690 qint64 avail = (self->device ? self->device->bytesAvailable() : self->buffer.size()) -
691 self->bufferStart;
692 return total > avail ? CborErrorUnexpectedEOF : CborNoError;
693}

695bool QCborStreamReaderPrivate::ensureStringIteration()
696{
697 if (currentElement.flags & CborIteratorFlag_IteratingStringChunks)
698 return true;
699
700 CborError err = cbor_value_begin_string_iteration(value: &currentElement);
701 if (!err)
702 return true;
703 handleError(err);
704 return false;
705}

707/*!
708 \internal
709 */
710inline void QCborStreamReader::preparse()
711{
712 if (lastError() == QCborError::NoError) {
713 type_ = cbor_value_get_type(value: &d->currentElement);
714
715 if (type_ == CborInvalidType) {
716 // We may have reached the end.
717 if (d->device && d->containerStack.isEmpty()) {
718 d->buffer.clear();
719 if (d->bufferStart)
720 d->device->skip(maxSize: d->bufferStart);
721 d->bufferStart = 0;
722 }
723 } else {
724 d->lastError = {};
725 // Undo the type mapping that TinyCBOR does (we have an explicit type
726 // for negative integer and we don't have separate types for Boolean,
727 // Null and Undefined).
728 if (type_ == CborBooleanType || type_ == CborNullType || type_ == CborUndefinedType) {
729 type_ = CborSimpleType;
730 value64 = quint8(d->buffer.at(i: d->bufferStart)) - CborSimpleType;
731 } else {
732 // Using internal TinyCBOR API!
733 value64 = _cbor_value_extract_int64_helper(value: &d->currentElement);
734
735 if (cbor_value_is_negative_integer(value: &d->currentElement))
736 type_ = quint8(QCborStreamReader::NegativeInteger);
737 }
738 }
739 } else {
740 type_ = Invalid;
741 }
742}

990/*!
991 Advance the CBOR stream decoding one element. You should usually call this
992 function when parsing fixed-width basic elements (that is, integers, simple
993 values, tags and floating point values). But this function can be called
994 when the current item is a string, array or map too and it will skip over
995 that entire element, including all contained elements.
996
997 This function returns true if advancing was successful, false otherwise. It
998 may fail if the stream is corrupt, incomplete or if the nesting level of
999 arrays and maps exceeds \a maxRecursion. Calling this function when
1000 hasNext() has returned false is also an error. If this function returns
1001 false, lastError() will return the error code detailing what the failure
1002 was.
1003
1004 \sa lastError(), isValid(), hasNext()
1005 */
1006bool QCborStreamReader::next(int maxRecursion)
1007{
1008 if (lastError() != QCborError::NoError)
1009 return false;
1010
1011 if (!hasNext()) {
1012 d->handleError(err: CborErrorAdvancePastEOF);
1013 } else if (maxRecursion < 0) {
1014 d->handleError(err: CborErrorNestingTooDeep);
1015 } else if (isContainer()) {
1016 // iterate over each element
1017 enterContainer();
1018 while (lastError() == QCborError::NoError && hasNext())
1019 next(maxRecursion: maxRecursion - 1);
1020 if (lastError() == QCborError::NoError)
1021 leaveContainer();
1022 } else if (isByteArray()) {
1023 char c;
1024 StringResult<qsizetype> r;
1025 do {
1026 r = readStringChunk(ptr: &c, maxlen: 1);
1027 } while (r.status == Ok);
1028 } else if (isString()) {
1029 // we need to use actual readString so we get UTF-8 validation
1030 StringResult<QString> r;
1031 do {
1032 r = readString();
1033 } while (r.status == Ok);
1034 } else {
1035 // fixed types
1036 CborError err = cbor_value_advance_fixed(it: &d->currentElement);
1037 if (err)
1038 d->handleError(err);
1039 }
1040
1041 preparse();
1042 return d->lastError == QCborError::NoError;
1043}

1392 would have returned. For strings, it returns the UTF-8 equivalent of the \l
1393 QString that would have been returned.

1396 For example:

1405 QCborError::InvalidUtf8String error, even when readString() does.
1406
1407 \sa currentStringChunkSize(), readString(), readByteArray(),
1408 isString(), isByteArray()
1409 */

1411QCborStreamReader::readStringChunk(char *ptr, qsizetype maxlen)
1412{
1413 auto r = d->readStringChunk(params: {ptr, maxlen});
1414 if (r.status == EndOfString && lastError() == QCborError::NoError)
1415 preparse();
1416 return r;
1417}

1434Q_NEVER_INLINE QCborStreamReader::StringResult<qsizetype>
1435QCborStreamReaderPrivate::readStringChunk(ReadStringChunk params)
1436{
1437 CborError err;
1438 size_t len;
1439 const void *content = nullptr;
1440 QCborStreamReader::StringResult<qsizetype> result;
1441 result.data = 0;
1442 result.status = QCborStreamReader::Error;
1443
1444 lastError = {};
1445 if (!ensureStringIteration())
1446 return result;
1447
1448 // Note: in the current implementation, the call into TinyCBOR below only
1449 // succeeds if we *already* have all the data in memory. That's obvious for
1450 // the case of direct memory (no QIODevice), whereas for QIODevices
1451 // qt_cbor_decoder_transfer_string() enforces that
1452 // QIODevice::bytesAvailable() be bigger than the amount we're about to
1453 // read.
1454#if 1
1455 // Using internal TinyCBOR API!
1456 err = _cbor_value_get_string_chunk(value: &currentElement, bufferptr: &content, len: &len, next: &currentElement);
1457#else
1458 // the above is effectively the same as:
1459 if (cbor_value_is_byte_string(&currentElement))
1460 err = cbor_value_get_byte_string_chunk(&currentElement, reinterpret_cast<const uint8_t **>(&content),
1461 &len, &currentElement);
1462 else
1463 err = cbor_value_get_text_string_chunk(&currentElement, reinterpret_cast<const char **>(&content),
1464 &len, &currentElement);
1465#endif
1466
1467 // Range check: using implementation-defined behavior in converting an
1468 // unsigned value out of range of the destination signed type (same as
1469 // "len > size_t(std::numeric_limits<qsizetype>::max())", but generates
1470 // better code with ICC and MSVC).
1471 if (!err && qsizetype(len) < 0)
1472 err = CborErrorDataTooLarge;
1473
1474 if (err) {
1475 if (err == CborErrorNoMoreStringChunks) {
1476 preread();
1477 err = cbor_value_finish_string_iteration(value: &currentElement);
1478 result.status = QCborStreamReader::EndOfString;
1479 }
1480 if (err)
1481 handleError(err);
1482 // caller musts call preparse()
1483 return result;
1484 }
1485
1486 qptrdiff offset = qptrdiff(content);
1487 bufferStart += offset;
1488 if (device) {
1489 // This first skip can't fail because we've already read this many bytes.
1490 device->skip(maxSize: bufferStart);
1491 }
1492
1493 if (params.isString()) {
1494 // readString()
1495 result.data = readStringChunk_unicode(params, utf8len: qsizetype(len));
1496 } else {
1497 // readByteArray() or readStringChunk()
1498 result.data = readStringChunk_byte(params, len: qsizetype(len));
1499 }
1500
1501 if (result.data < 0)
1502 return result; // error
1503
1504 // adjust the buffers after we're done reading the string
1505 bufferStart += len;
1506 if (device) {
1507 qsizetype remainingInBuffer = buffer.size() - bufferStart;
1508
1509 if (remainingInBuffer <= 0) {
1510 // We've read from the QIODevice more than what was in the buffer.
1511 buffer.truncate(pos: 0);
1512 } else {
1513 // There's still data buffered, but we need to move it around.
1514 char *ptr = buffer.data();
1515 memmove(dest: ptr, src: ptr + bufferStart, n: remainingInBuffer);
1516 buffer.truncate(pos: remainingInBuffer);
1517 }
1518
1519 bufferStart = 0;
1520 }
1521
1522 preread();
1523 result.status = QCborStreamReader::Ok;
1524 return result;
1525}

1527inline qsizetype
1528QCborStreamReaderPrivate::readStringChunk_byte(ReadStringChunk params, qsizetype len)
1529{
1530 qint64 actuallyRead;
1531 qsizetype toRead = qsizetype(len);
1532 qsizetype left = 0; // bytes from the chunk not copied to the user buffer, to discard
1533 char *ptr = nullptr;
1534
1535 if (params.isPlainPointer()) {
1536 left = toRead - params.maxlen_or_type;
1537 if (left < 0)
1538 left = 0; // buffer bigger than string
1539 else
1540 toRead = params.maxlen_or_type; // buffer smaller than string
1541 ptr = params.ptr;
1542 } else if (params.isByteArray()) {
1543 // See note above on having ensured there is enough incoming data.
1544 auto oldSize = params.array->size();
1545 auto newSize = oldSize;
1546 if (add_overflow<decltype(newSize)>(v1: oldSize, v2: toRead, r: &newSize)) {
1547 handleError(err: CborErrorDataTooLarge);
1548 return -1;
1549 }
1550 QT_TRY {
1551 params.array->resize(size: newSize);
1552 } QT_CATCH (const std::bad_alloc &) {
1553 // the distinction between DataTooLarge and OOM is mostly for
1554 // compatibility with Qt 5; in Qt 6, we could consider everything
1555 // to be OOM.
1556 handleError(err: newSize > MaxByteArraySize ? CborErrorDataTooLarge: CborErrorOutOfMemory);
1557 return -1;
1558 }
1559
1560 ptr = const_cast<char *>(params.array->constData()) + oldSize;
1561 }
1562
1563 if (device) {
1564 actuallyRead = device->read(data: ptr, maxlen: toRead);
1565
1566 if (actuallyRead != toRead) {
1567 actuallyRead = -1;
1568 } else if (left) {
1569 qint64 skipped = device->skip(maxSize: left);
1570 if (skipped != left)
1571 actuallyRead = -1;
1572 }
1573
1574 if (actuallyRead < 0) {
1575 handleError(err: CborErrorIO);
1576 return -1;
1577 }
1578 } else {
1579 actuallyRead = toRead;
1580 memcpy(dest: ptr, src: buffer.constData() + bufferStart, n: toRead);
1581 }
1582
1583 return actuallyRead;
1584}

1586inline qsizetype
1587QCborStreamReaderPrivate::readStringChunk_unicode(ReadStringChunk params, qsizetype utf8len)
1588{
1589 // See QUtf8::convertToUnicode() a detailed explanation of why this
1590 // conversion uses the same number of words or less.
1591 QChar *begin = nullptr;
1592 if (params.isString()) {
1593 QT_TRY {
1594 params.string->resize(size: utf8len);
1595 } QT_CATCH (const std::bad_alloc &) {
1596 if (utf8len > MaxStringSize)
1597 handleError(err: CborErrorDataTooLarge);
1598 else
1599 handleError(err: CborErrorOutOfMemory);
1600 return -1;
1601 }
1602
1603 begin = const_cast<QChar *>(params.string->constData());
1604 }
1605
1606 QChar *ptr = begin;
1607 QStringConverter::State cs(QStringConverter::Flag::Stateless);
1608 if (device == nullptr) {
1609 // Easy case: we can decode straight from the buffer we already have
1610 ptr = QUtf8::convertToUnicode(out: ptr, in: { buffer.constData() + bufferStart, utf8len }, state: &cs);
1611 } else {
1612 // read in chunks, to avoid creating large, intermediate buffers
1613 constexpr qsizetype StringChunkSize = 16384;
1614 qsizetype chunkSize = qMin(a: StringChunkSize, b: utf8len);
1615 QVarLengthArray<char> chunk(chunkSize);
1616
1617 cs = { QStringConverter::Flag::ConvertInitialBom };
1618 while (utf8len > 0 && cs.invalidChars == 0) {
1619 qsizetype toRead = qMin(a: chunkSize, b: utf8len);
1620 qint64 actuallyRead = device->read(data: chunk.data(), maxlen: toRead);
1621 if (actuallyRead == toRead)
1622 ptr = QUtf8::convertToUnicode(out: ptr, in: { chunk.data(), toRead }, state: &cs);
1623
1624 if (actuallyRead != toRead) {
1625 handleError(err: CborErrorIO);
1626 return -1;
1627 }
1628 utf8len -= toRead;
1629 }
1630 }
1631
1632 if (cs.invalidChars != 0 || cs.remainingChars != 0) {
1633 handleError(err: CborErrorInvalidUtf8TextString);
1634 return -1;
1635 }
1636
1637 qsizetype size = ptr - begin;
1638 if (params.isString())
1639 params.string->truncate(pos: size);
1640 return size;
1641}
