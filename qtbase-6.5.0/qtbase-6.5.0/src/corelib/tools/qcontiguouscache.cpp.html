31/*! \class QContiguousCache
32 \inmodule QtCore
33 \brief The QContiguousCache class is a template class that provides a contiguous cache.
34 \ingroup tools
35 \ingroup shared
36 \reentrant
37 \since 4.6
38
39 The QContiguousCache class provides an efficient way of caching items for
40 display in a user interface view. Unlike QCache, it adds a restriction
41 that elements within the cache are contiguous. This has the advantage
42 of matching how user interface views most commonly request data, as
43 a set of rows localized around the current scrolled position. This
44 restriction allows the cache to consume less memory and processor
45 cycles than QCache.
46
47 QContiguousCache operates on a fixed capacity, set with setCapacity() or
48 passed as a parameter to the constructor. This capacity is the upper bound
49 on memory usage by the cache itself, not including the memory allocated by
50 the elements themselves. Note that a cache with a capacity of zero (the
51 default) means no items will be stored: the insert(), append() and
52 prepend() operations will effectively be no-ops. Therefore, it's important
53 to set the capacity to a reasonable value before adding items to the cache.
54
55 The simplest way of using a contiguous cache is to use the append()
56 and prepend().
57
58 \snippet code/src_corelib_tools_qcontiguouscache.cpp 0
59
60 If the cache is full then the item at the opposite end of the cache from
61 where the new item is appended or prepended will be removed.
62
63 This usage can be further optimized by using the insert() function
64 in the case where the requested row is a long way from the currently cached
65 items. If there is a gap between where the new item is inserted and the currently
66 cached items then the existing cached items are first removed to retain
67 the contiguous nature of the cache. Hence it is important to take some care then
68 when using insert() in order to avoid unwanted clearing of the cache.
69
70 The range of valid indexes for the QContiguousCache class are from
71 0 to INT_MAX. Calling prepend() such that the first index would become less
72 than 0 or append() such that the last index would become greater
73 than INT_MAX can result in the indexes of the cache being invalid.
74 When the cache indexes are invalid it is important to call
75 normalizeIndexes() before calling any of containsIndex(), firstIndex(),
76 lastIndex(), at() or \l{QContiguousCache::operator[]()}{operator[]()}.
77 Calling these functions when the cache has invalid indexes will result in
78 undefined behavior. The indexes can be checked by using areIndexesValid()
79
80 In most cases the indexes will not exceed 0 to INT_MAX, and
81 normalizeIndexes() will not need to be used.
82
83 See the \l{Contiguous Cache Example}{Contiguous Cache} example.
84*/

252/*! \fn template<typename T> const T &QContiguousCache<T>::at(qsizetype i) const
253
254 Returns the item at index position \a i in the cache. \a i must
255 be a valid index position in the cache (i.e, firstIndex() <= \a i <= lastIndex()).
256
257 The indexes in the cache refer to the number of positions the item is from the
258 first item appended into the cache. That is to say a cache with a capacity of
259 100, that has had 150 items appended will have a valid index range of
260 50 to 149. This allows inserting and retrieving items into the cache based
261 on a theoretical infinite list
262
263 \sa firstIndex(), lastIndex(), insert(), operator[]()
264*/

304/*! \fn template<typename T> void QContiguousCache<T>::insert(qsizetype i, const T &value)
305
306 Inserts the \a value at the index position \a i. If the cache already contains
307 an item at \a i then that value is replaced. If \a i is either one more than
308 lastIndex() or one less than firstIndex() it is the equivalent to an append()
309 or a prepend().
310
311 If the given index \a i is not within the current range of the cache nor adjacent
312 to the bounds of the cache's index range, the cache is first cleared before
313 inserting the item. At this point the cache will have a size of 1. It is
314 worthwhile taking effort to insert items in an order that starts adjacent
315 to the current index range for the cache.
316
317 The range of valid indexes for the QContiguousCache class are from
318 0 to INT_MAX. Inserting outside of this range has undefined behavior.
319
320
321 \sa prepend(), append(), isFull(), firstIndex(), lastIndex()
322*/

423/*! \fn template<typename T> bool QContiguousCache<T>::areIndexesValid() const
