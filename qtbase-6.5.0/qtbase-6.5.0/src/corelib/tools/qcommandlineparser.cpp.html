108/*!
109 \since 5.2
110 \class QCommandLineParser
111 \inmodule QtCore
112 \ingroup tools
113
114 \brief The QCommandLineParser class provides a means for handling the
115 command line options.
116
117 QCoreApplication provides the command-line arguments as a simple list of strings.
118 QCommandLineParser provides the ability to define a set of options, parse the
119 command-line arguments, and store which options have actually been used, as
120 well as option values.
121
122 Any argument that isn't an option (i.e. doesn't start with a \c{-}) is stored
123 as a "positional argument".
124
125 The parser handles short names, long names, more than one name for the same
126 option, and option values.
127
128 Options on the command line are recognized as starting with a single or
129 double \c{-} character(s).
130 The option \c{-} (single dash alone) is a special case, often meaning standard
131 input, and not treated as an option. The parser will treat everything after the
132 option \c{--} (double dash) as positional arguments.
133
134 Short options are single letters. The option \c{v} would be specified by
135 passing \c{-v} on the command line. In the default parsing mode, short options
136 can be written in a compact form, for instance \c{-abc} is equivalent to \c{-a -b -c}.
137 The parsing mode for can be set to ParseAsLongOptions, in which case \c{-abc}
138 will be parsed as the long option \c{abc}.
139
140 Long options are more than one letter long and cannot be compacted together.
141 The long option \c{verbose} would be passed as \c{--verbose} or \c{-verbose}.
142
143 Passing values to options can be done using the assignment operator: \c{-v=value}
144 \c{--verbose=value}, or a space: \c{-v value} \c{--verbose value}, i.e. the next
145 argument is used as value (even if it starts with a \c{-}).
146
147 The parser does not support optional values - if an option is set to
148 require a value, one must be present. If such an option is placed last
149 and has no value, the option will be treated as if it had not been
150 specified.
151
152 The parser does not automatically support negating or disabling long options
153 by using the format \c{--disable-option} or \c{--no-option}. However, it is
154 possible to handle this case explicitly by making an option with \c{no-option}
155 as one of its names, and handling the option explicitly.
156
157 Example:
158 \snippet code/src_corelib_tools_qcommandlineparser_main.cpp 0
159
160 If your compiler supports the C++11 standard, the three addOption() calls in
161 the above example can be simplified:
162 \snippet code/src_corelib_tools_qcommandlineparser_main.cpp cxx11
163
164 Known limitation: the parsing of Qt options inside QCoreApplication and subclasses
165 happens before QCommandLineParser exists, so it can't take it into account. This
166 means any option value that looks like a builtin Qt option, will be treated by
167 QCoreApplication as a builtin Qt option. Example: \c{--profile -reverse} will
168 lead to QGuiApplication seeing the -reverse option set, and removing it from
169 QCoreApplication::arguments() before QCommandLineParser defines the \c{profile}
170 option and parses the command line.
171
172 \section2 How to Use QCommandLineParser in Complex Applications
173
174 In practice, additional error checking needs to be performed on the positional
175 arguments and option values. For example, ranges of numbers should be checked.
176
177 It is then advisable to introduce a function to do the command line parsing
178 which takes a struct or class receiving the option values returning an
179 enumeration representing the result. The dnslookup example of the QtNetwork
180 module illustrates this:
181
182 \snippet dnslookup.h 0
183
184 \snippet dnslookup.cpp 0
185
186 In the main function, help should be printed to the standard output if the help option
187 was passed and the application should return the exit code 0.
188
189 If an error was detected, the error message should be printed to the standard
190 error output and the application should return an exit code other than 0.
191
192 \snippet dnslookup.cpp 1
193
194 A special case to consider here are GUI applications on Windows and mobile
195 platforms. These applications may not use the standard output or error channels
196 since the output is either discarded or not accessible.
197
198 On Windows, QCommandLineParser uses message boxes to display usage information
199 and errors if no console window can be obtained.
200
201 For other platforms, it is recommended to display help texts and error messages
202 using a QMessageBox. To preserve the formatting of the help text, rich text
203 with \c <pre> elements should be used:
204
205 \code
206
207 switch (parseCommandLine(parser, &query, &errorMessage)) {
208 case CommandLineOk:
209 break;
210 case CommandLineError:
211 QMessageBox::warning(0, QGuiApplication::applicationDisplayName(),
212 "<html><head/><body><h2>" + errorMessage + "</h2><pre>"
213 + parser.helpText() + "</pre></body></html>");
214 return 1;
215 case CommandLineVersionRequested:
216 QMessageBox::information(0, QGuiApplication::applicationDisplayName(),
217 QGuiApplication::applicationDisplayName() + ' '
218 + QCoreApplication::applicationVersion());
219 return 0;
220 case CommandLineHelpRequested:
221 QMessageBox::warning(0, QGuiApplication::applicationDisplayName(),
222 "<html><head/><body><pre>"
223 + parser.helpText() + "</pre></body></html>");
224 return 0;
225 }
226 \endcode
227
228 However, this does not apply to the dnslookup example, because it is a
229 console application.
230
231 \sa QCommandLineOption, QCoreApplication
232*/

250/*!
251 \enum QCommandLineParser::SingleDashWordOptionMode
252
253 This enum describes the way the parser interprets command-line
254 options that use a single dash followed by multiple letters, as as \c{-abc}.
255
256 \value ParseAsCompactedShortOptions \c{-abc} is interpreted as \c{-a -b -c},
257 i.e. as three short options that have been compacted on the command-line,
258 if none of the options take a value. If \c{a} takes a value, then it
259 is interpreted as \c{-a bc}, i.e. the short option \c{a} followed by the value \c{bc}.
260 This is typically used in tools that behave like compilers, in order
261 to handle options such as \c{-DDEFINE=VALUE} or \c{-I/include/path}.
262 This is the default parsing mode. New applications are recommended to
263 use this mode.
264
265 \value ParseAsLongOptions \c{-abc} is interpreted as \c{--abc},
266 i.e. as the long option named \c{abc}. This is how Qt's own tools
267 (uic, rcc...) have always been parsing arguments. This mode should be
268 used for preserving compatibility in applications that were parsing
269 arguments in such a way. There is an exception if the \c{a} option has the
270 QCommandLineOption::ShortOptionStyle flag set, in which case it is still
271 interpreted as \c{-a bc}.
272
273 \sa setSingleDashWordOptionMode()
274*/
275

285/*!
286 \enum QCommandLineParser::OptionsAfterPositionalArgumentsMode
287
288 This enum describes the way the parser interprets options that
289 occur after positional arguments.
290
291 \value ParseAsOptions \c{application argument --opt -t} is interpreted as setting
292 the options \c{opt} and \c{t}, just like \c{application --opt -t argument} would do.
293 This is the default parsing mode. In order to specify that \c{--opt} and \c{-t}
294 are positional arguments instead, the user can use \c{--}, as in
295 \c{application argument -- --opt -t}.
296
297 \value ParseAsPositionalArguments \c{application argument --opt} is interpreted as
298 having two positional arguments, \c{argument} and \c{--opt}.
299 This mode is useful for executables that aim to launch other executables
300 (e.g. wrappers, debugging tools, etc.) or that support internal commands
301 followed by options for the command. \c{argument} is the name of the command,
302 and all options occurring after it can be collected and parsed by another
303 command line parser, possibly in another executable.
304
305 \sa setOptionsAfterPositionalArgumentsMode()
306
307 \since 5.6
308*/
309

320/*!
321 Adds the option \a option to look for while parsing.
322
323 Returns \c true if adding the option was successful; otherwise returns \c false.
324
325 Adding the option fails if there is no name attached to the option, or
326 the option has a name that clashes with an option name added before.
327 */
328bool QCommandLineParser::addOption(const QCommandLineOption &option)
329{
330 const QStringList optionNames = option.names();
331
332 if (!optionNames.isEmpty()) {
333 for (const QString &name : optionNames) {
334 if (d->nameHash.contains(key: name)) {
335 qWarning() << "QCommandLineParser: already having an option named" << name;
336 return false;
337 }
338 }
339
340 d->commandLineOptionList.append(t: option);
341
342 const int offset = d->commandLineOptionList.size() - 1;
343 for (const QString &name : optionNames)
344 d->nameHash.insert(key: name, value: offset);
345
346 return true;
347 }
348
349 return false;
350}

352/*!
353 \since 5.4
354
355 Adds the options to look for while parsing. The options are specified by
356 the parameter \a options.
357
358 Returns \c true if adding all of the options was successful; otherwise
359 returns \c false.
360
361 See the documentation for addOption() for when this function may fail.
362*/
363bool QCommandLineParser::addOptions(const QList<QCommandLineOption> &options)
364{
365 // should be optimized (but it's no worse than what was possible before)
366 bool result = true;
367 for (QList<QCommandLineOption>::const_iterator it = options.begin(), end = options.end(); it != end; ++it)
368 result &= addOption(option: *it);
369 return result;
370}

471/*!
472 Parses the command line \a arguments.
473
474 Most programs don't need to call this, a simple call to process() is enough.
475
476 parse() is more low-level, and only does the parsing. The application will have to
477 take care of the error handling, using errorText() if parse() returns \c false.

483

485
486 Returns \c false in case of a parse error (unknown option or missing value); returns \c true otherwise.
487
488 \sa process()
489*/
490bool QCommandLineParser::parse(const QStringList &arguments)
491{
492 return d->parse(args: arguments);
493}

545/*!
546 Processes the command line \a arguments.
547
548 In addition to parsing the options (like parse()), this function also handles the builtin
549 options and handles errors.
550
551 The builtin options are \c{--version} if addVersionOption was called and
552 \c{--help} / \c{--help-all} if addHelpOption was called.
553
554 When invoking one of these options, or when an error happens (for instance an unknown option was
555 passed), the current process will then stop, using the exit() function.
556
557 \sa QCoreApplication::arguments(), parse()
558 */
559void QCommandLineParser::process(const QStringList &arguments)
560{
561 if (!d->parse(args: arguments)) {
562 showParserMessage(message: QCoreApplication::applicationName() + ": "_L1 + errorText() + u'\n', type: ErrorMessage);
563 qt_call_post_routines();
564 ::exit(EXIT_FAILURE);
565 }
566
567 if (d->builtinVersionOption && isSet(QStringLiteral("version")))
568 showVersion();
569
570 if (d->builtinHelpOption && isSet(QStringLiteral("help")))
571 d->showHelp(EXIT_SUCCESS, includeQtOptions: false);
572
573 if (d->builtinHelpOption && isSet(QStringLiteral("help-all")))
574 d->showHelp(EXIT_SUCCESS, includeQtOptions: true);
575}

611/*!
612 \internal
613 \brief Parse the value for a given option, if it was defined to expect one.
614
615 The value is taken from the next argument, or after the equal sign in \a argument.
616
617 \param optionName the short option name
618 \param argument the argument from the command line currently parsed. Only used for -k=value parsing.
619 \param argumentIterator iterator to the currently parsed argument. Incremented if the next argument contains the value.
620 \param argsEnd args.end(), to check if ++argumentIterator goes out of bounds
621 Returns \c true on success.
622 */
623bool QCommandLineParserPrivate::parseOptionValue(const QString &optionName, const QString &argument,
624 QStringList::const_iterator *argumentIterator, QStringList::const_iterator argsEnd)
625{
626 const QLatin1Char assignChar('=');
627 const NameHash_t::const_iterator nameHashIt = nameHash.constFind(key: optionName);
628 if (nameHashIt != nameHash.constEnd()) {
629 const int assignPos = argument.indexOf(c: assignChar);
630 const NameHash_t::mapped_type optionOffset = *nameHashIt;
631 const bool withValue = !commandLineOptionList.at(i: optionOffset).valueName().isEmpty();
632 if (withValue) {
633 if (assignPos == -1) {
634 ++(*argumentIterator);
635 if (*argumentIterator == argsEnd) {
636 errorText = QCommandLineParser::tr(sourceText: "Missing value after '%1'.").arg(a: argument);
637 return false;
638 }
639 optionValuesHash[optionOffset].append(t: *(*argumentIterator));
640 } else {
641 optionValuesHash[optionOffset].append(t: argument.mid(position: assignPos + 1));
642 }
643 } else {
644 if (assignPos != -1) {
645 errorText = QCommandLineParser::tr(sourceText: "Unexpected value after '%1'.").arg(a: argument.left(n: assignPos));
646 return false;
647 }
648 }
649 }
650 return true;
651}

653/*!
654 \internal
655
656 Parse the list of arguments \a args, and fills in
657 optionNames, optionValuesHash, unknownOptionNames, positionalArguments, and errorText.
658
659 Any results from a previous parse operation are removed.
660
661 The parser will not look for further options once it encounters the option
662 \c{--}; this does not include when \c{--} follows an option that requires a value.
663 */
664bool QCommandLineParserPrivate::parse(const QStringList &args)
665{
666 needsParsing = false;
667 bool error = false;
668
669 const QString doubleDashString(QStringLiteral("--"));
670 const QLatin1Char dashChar('-');
671 const QLatin1Char assignChar('=');
672
673 bool forcePositional = false;
674 errorText.clear();
675 positionalArgumentList.clear();
676 optionNames.clear();
677 unknownOptionNames.clear();
678 optionValuesHash.clear();
679
680 if (args.isEmpty()) {
681 qWarning(msg: "QCommandLineParser: argument list cannot be empty, it should contain at least the executable name");
682 return false;
683 }
684
685 QStringList::const_iterator argumentIterator = args.begin();
686 ++argumentIterator; // skip executable name
687
688 for (; argumentIterator != args.end() ; ++argumentIterator) {
689 QString argument = *argumentIterator;
690
691 if (forcePositional) {
692 positionalArgumentList.append(t: argument);
693 } else if (argument.startsWith(s: doubleDashString)) {
694 if (argument.size() > 2) {
695 QString optionName = argument.mid(position: 2).section(asep: assignChar, astart: 0, aend: 0);
696 if (registerFoundOption(optionName)) {
697 if (!parseOptionValue(optionName, argument, argumentIterator: &argumentIterator, argsEnd: args.end()))
698 error = true;
699 } else {
700 error = true;
701 }
702 } else {
703 forcePositional = true;
704 }
705 } else if (argument.startsWith(c: dashChar)) {
706 if (argument.size() == 1) { // single dash ("stdin")
707 positionalArgumentList.append(t: argument);
708 continue;
709 }
710 switch (singleDashWordOptionMode) {
711 case QCommandLineParser::ParseAsCompactedShortOptions:
712 {
713 QString optionName;
714 bool valueFound = false;
715 for (int pos = 1 ; pos < argument.size(); ++pos) {
716 optionName = argument.mid(position: pos, n: 1);
717 if (!registerFoundOption(optionName)) {
718 error = true;
719 } else {
720 const NameHash_t::const_iterator nameHashIt = nameHash.constFind(key: optionName);
721 Q_ASSERT(nameHashIt != nameHash.constEnd()); // checked by registerFoundOption
722 const NameHash_t::mapped_type optionOffset = *nameHashIt;
723 const bool withValue = !commandLineOptionList.at(i: optionOffset).valueName().isEmpty();
724 if (withValue) {
725 if (pos + 1 < argument.size()) {
726 if (argument.at(i: pos + 1) == assignChar)
727 ++pos;
728 optionValuesHash[optionOffset].append(t: argument.mid(position: pos + 1));
729 valueFound = true;
730 }
731 break;
732 }
733 if (pos + 1 < argument.size() && argument.at(i: pos + 1) == assignChar)
734 break;
735 }
736 }
737 if (!valueFound && !parseOptionValue(optionName, argument, argumentIterator: &argumentIterator, argsEnd: args.end()))
738 error = true;
739 break;
740 }
741 case QCommandLineParser::ParseAsLongOptions:
742 {
743 if (argument.size() > 2) {
744 const QString possibleShortOptionStyleName = argument.mid(position: 1, n: 1);
745 const auto shortOptionIt = nameHash.constFind(key: possibleShortOptionStyleName);
746 if (shortOptionIt != nameHash.constEnd()) {
747 const auto &arg = commandLineOptionList.at(i: *shortOptionIt);
748 if (arg.flags() & QCommandLineOption::ShortOptionStyle) {
749 registerFoundOption(optionName: possibleShortOptionStyleName);
750 optionValuesHash[*shortOptionIt].append(t: argument.mid(position: 2));
751 break;
752 }
753 }
754 }
755 const QString optionName = argument.mid(position: 1).section(asep: assignChar, astart: 0, aend: 0);
756 if (registerFoundOption(optionName)) {
757 if (!parseOptionValue(optionName, argument, argumentIterator: &argumentIterator, argsEnd: args.end()))
758 error = true;
759 } else {
760 error = true;
761 }
762 break;
763 }
764 }
765 } else {
766 positionalArgumentList.append(t: argument);
767 if (optionsAfterPositionalArgumentsMode == QCommandLineParser::ParseAsPositionalArguments)
768 forcePositional = true;
769 }
770 if (argumentIterator == args.end())
771 break;
772 }
773 return !error;
774}

1027static QString wrapText(const QString &names, int optionNameMaxWidth, const QString &description)
1028{
1029 const auto nl = u'\n';
1030 const auto indentation = " "_L1;
1031
1032 // In case the list of option names is very long, wrap it as well
1033 int nameIndex = 0;
1034 auto nextNameSection = [&]() {
1035 QString section = names.mid(position: nameIndex, n: optionNameMaxWidth);
1036 nameIndex += section.size();
1037 return section;
1038 };
1039
1040 QString text;
1041 int lineStart = 0;
1042 int lastBreakable = -1;
1043 const int max = 79 - (indentation.size() + optionNameMaxWidth + 1);
1044 int x = 0;
1045 const int len = description.size();
1046
1047 for (int i = 0; i < len; ++i) {
1048 ++x;
1049 const QChar c = description.at(i);
1050 if (c.isSpace())
1051 lastBreakable = i;
1052
1053 int breakAt = -1;
1054 int nextLineStart = -1;
1055 if (x > max && lastBreakable != -1) {
1056 // time to break and we know where
1057 breakAt = lastBreakable;
1058 nextLineStart = lastBreakable + 1;
1059 } else if ((x > max - 1 && lastBreakable == -1) || i == len - 1) {
1060 // time to break but found nowhere [-> break here], or end of last line
1061 breakAt = i + 1;
1062 nextLineStart = breakAt;
1063 } else if (c == nl) {
1064 // forced break
1065 breakAt = i;
1066 nextLineStart = i + 1;
1067 }
1068
1069 if (breakAt != -1) {
1070 const int numChars = breakAt - lineStart;
1071 //qDebug() << "breakAt=" << description.at(breakAt) << "breakAtSpace=" << breakAtSpace << lineStart << "to" << breakAt << description.mid(lineStart, numChars);
1072 text += indentation + nextNameSection().leftJustified(width: optionNameMaxWidth) + u' ';
1073 text += QStringView{description}.mid(pos: lineStart, n: numChars) + nl;
1074 x = 0;
1075 lastBreakable = -1;
1076 lineStart = nextLineStart;
1077 if (lineStart < len && description.at(i: lineStart).isSpace())
1078 ++lineStart; // don't start a line with a space
1079 i = lineStart;
1080 }
1081 }
1082
1083 while (nameIndex < names.size()) {
1084 text += indentation + nextNameSection() + nl;
1085 }
1086
1087 return text;
1088}

1090QString QCommandLineParserPrivate::helpText(bool includeQtOptions) const
1091{
1092 const QLatin1Char nl('\n');
1093 QString text;
1094 QString usage;
1095 // executable name
1096 usage += qApp ? QCoreApplication::arguments().constFirst() : QStringLiteral("<executable_name>");
1097 QList<QCommandLineOption> options = commandLineOptionList;
1098 if (includeQtOptions && qApp)
1099 qApp->d_func()->addQtOptions(options: &options);
1100 if (!options.isEmpty())
1101 usage += u' ' + QCommandLineParser::tr(sourceText: "[options]");
1102 for (const PositionalArgumentDefinition &arg : positionalArgumentDefinitions)
1103 usage += u' ' + arg.syntax;
1104 text += QCommandLineParser::tr(sourceText: "Usage: %1").arg(a: usage) + nl;
1105 if (!description.isEmpty())
1106 text += description + nl;
1107 text += nl;
1108 if (!options.isEmpty())
1109 text += QCommandLineParser::tr(sourceText: "Options:") + nl;
1110 QStringList optionNameList;
1111 optionNameList.reserve(size: options.size());
1112 int longestOptionNameString = 0;
1113 for (const QCommandLineOption &option : std::as_const(t&: options)) {
1114 if (option.flags() & QCommandLineOption::HiddenFromHelp)
1115 continue;
1116 const QStringList optionNames = option.names();
1117 QString optionNamesString;
1118 for (const QString &optionName : optionNames) {
1119 const int numDashes = optionName.size() == 1 ? 1 : 2;
1120 optionNamesString += QLatin1StringView("--", numDashes) + optionName + ", "_L1;
1121 }
1122 if (!optionNames.isEmpty())
1123 optionNamesString.chop(n: 2); // remove trailing ", "
1124 const auto valueName = option.valueName();
1125 if (!valueName.isEmpty())
1126 optionNamesString += " <"_L1 + valueName + u'>';
1127 optionNameList.append(t: optionNamesString);
1128 longestOptionNameString = qMax(a: longestOptionNameString, b: optionNamesString.size());
1129 }
1130 ++longestOptionNameString;
1131 const int optionNameMaxWidth = qMin(a: 50, b: longestOptionNameString);
1132 auto optionNameIterator = optionNameList.cbegin();
1133 for (const QCommandLineOption &option : std::as_const(t&: options)) {
1134 if (option.flags() & QCommandLineOption::HiddenFromHelp)
1135 continue;
1136 text += wrapText(names: *optionNameIterator, optionNameMaxWidth, description: option.description());
1137 ++optionNameIterator;
1138 }
1139 if (!positionalArgumentDefinitions.isEmpty()) {
1140 if (!options.isEmpty())
1141 text += nl;
1142 text += QCommandLineParser::tr(sourceText: "Arguments:") + nl;
1143 for (const PositionalArgumentDefinition &arg : positionalArgumentDefinitions)
1144 text += wrapText(names: arg.name, optionNameMaxWidth, description: arg.description);
1145 }
1146 return text;
1147}
