19/*!
20 \class QSemaphore
21 \inmodule QtCore
22 \brief The QSemaphore class provides a general counting semaphore.
23
24 \threadsafe
25
26 \ingroup thread
27
28 A semaphore is a generalization of a mutex. While a mutex can
29 only be locked once, it's possible to acquire a semaphore
30 multiple times. Semaphores are typically used to protect a
31 certain number of identical resources.
32
33 Semaphores support two fundamental operations, acquire() and
34 release():
35
36 \list
37 \li acquire(\e{n}) tries to acquire \e n resources. If there aren't
38 that many resources available, the call will block until this
39 is the case.
40 \li release(\e{n}) releases \e n resources.
41 \endlist
42
43 There's also a tryAcquire() function that returns immediately if
44 it cannot acquire the resources, and an available() function that
45 returns the number of available resources at any time.
46
47 Example:
48
49 \snippet code/src_corelib_thread_qsemaphore.cpp 0
50
51 A typical application of semaphores is for controlling access to
52 a circular buffer shared by a producer thread and a consumer
53 thread. The \l{Semaphores Example} shows how
54 to use QSemaphore to solve that problem.
55
56 A non-computing example of a semaphore would be dining at a
57 restaurant. A semaphore is initialized with the number of chairs
58 in the restaurant. As people arrive, they want a seat. As seats
59 are filled, available() is decremented. As people leave, the
60 available() is incremented, allowing more people to enter. If a
61 party of 10 people want to be seated, but there are only 9 seats,
62 those 10 people will wait, but a party of 4 people would be
63 seated (taking the available seats to 5, making the party of 10
64 people wait longer).
65
66 \sa QSemaphoreReleaser, QMutex, QWaitCondition, QThread, {Semaphores Example}
67*/
68
69/*
70 QSemaphore futex operation
71
72 QSemaphore stores a 32-bit integer with the counter of currently available
73 tokens (value between 0 and INT_MAX). When a thread attempts to acquire n
74 tokens and the counter is larger than that, we perform a compare-and-swap
75 with the new count. If that succeeds, the acquisition worked; if not, we
76 loop again because the counter changed. If there were not enough tokens,
77 we'll perform a futex-wait.
78
79 Before we do, we set the high bit in the futex to indicate that semaphore
80 is contended: that is, there's a thread waiting for more tokens. On
81 release() for n tokens, we perform a fetch-and-add of n and then check if
82 that high bit was set. If it was, then we clear that bit and perform a
83 futex-wake on the semaphore to indicate the waiting threads can wake up and
84 acquire tokens. Which ones get woken up is unspecified.
85
86 If the system has the ability to wake up a precise number of threads, has
87 Linux's FUTEX_WAKE_OP functionality, and is 64-bit, instead of using a
88 single bit indicating a contended semaphore, we'll store the number of
89 tokens *plus* total number of waiters in the high word. Additionally, all
90 multi-token waiters will be waiting on that high word. So when releasing n
91 tokens on those systems, we tell the kernel to wake up n single-token
92 threads and all of the multi-token ones. Which threads get woken up is
93 unspecified, but it's likely single-token threads will get woken up first.
94 */
95
96#if defined(FUTEX_OP) && QT_POINTER_SIZE > 4
97static constexpr bool futexHasWaiterCount = true;
98#else
99static constexpr bool futexHasWaiterCount = false;
100#endif
101
102static constexpr quintptr futexNeedsWakeAllBit = futexHasWaiterCount ?
103 (Q_UINT64_C(1) << (sizeof(quintptr) * CHAR_BIT - 1)) : 0x80000000U;
104
105static int futexAvailCounter(quintptr v)
106{
107 // the low 31 bits
108 if (futexHasWaiterCount) {
109 // the high bit of the low word isn't used
110 Q_ASSERT((v & 0x80000000U) == 0);
111
112 // so we can be a little faster
113 return int(unsigned(v));
114 }
115 return int(v & 0x7fffffffU);
116}
117
118static bool futexNeedsWake(quintptr v)
119{
120 // If we're counting waiters, the number of waiters plus value is stored in the
121 // low 31 bits of the high word (that is, bits 32-62). If we're not, then we only
122 // use futexNeedsWakeAllBit to indicate anyone is waiting.
123 if constexpr (futexHasWaiterCount)
124 return unsigned(quint64(v) >> 32) > unsigned(v);
125 return v >> 31;
126}
127
128static QBasicAtomicInteger<quint32> *futexLow32(QBasicAtomicInteger<quintptr> *ptr)
129{
130 auto result = reinterpret_cast<QBasicAtomicInteger<quint32> *>(ptr);
131#if Q_BYTE_ORDER == Q_BIG_ENDIAN && QT_POINTER_SIZE > 4
132 ++result;
133#endif
134 return result;
135}
136
137static QBasicAtomicInteger<quint32> *futexHigh32(QBasicAtomicInteger<quintptr> *ptr)
138{
139 Q_ASSERT(futexHasWaiterCount);
140 auto result = reinterpret_cast<QBasicAtomicInteger<quint32> *>(ptr);
141#if Q_BYTE_ORDER == Q_LITTLE_ENDIAN && QT_POINTER_SIZE > 4
142 ++result;
143#endif
144 return result;
145}

147template <bool IsTimed> bool
148futexSemaphoreTryAcquire_loop(QBasicAtomicInteger<quintptr> &u, quintptr curValue, quintptr nn, int timeout)
149{
150 QDeadlineTimer timer(IsTimed ? QDeadlineTimer(timeout) : QDeadlineTimer());
151 qint64 remainingTime = timeout * Q_INT64_C(1000) * 1000;
152 int n = int(unsigned(nn));
153
154 // we're called after one testAndSet, so start by waiting first
155 for (;;) {
156 // indicate we're waiting
157 auto ptr = futexLow32(ptr: &u);
158 if (n > 1 || !futexHasWaiterCount) {
159 u.fetchAndOrRelaxed(valueToAdd: futexNeedsWakeAllBit);
160 curValue |= futexNeedsWakeAllBit;
161 if constexpr (futexHasWaiterCount) {
162 Q_ASSERT(n > 1);
163 ptr = futexHigh32(ptr: &u);
164 curValue = quint64(curValue) >> 32;
165 }
166 }
167
168 if (IsTimed && remainingTime > 0) {
169 bool timedout = !futexWait(futex&: *ptr, expectedValue: curValue, nstimeout: remainingTime);
170 if (timedout)
171 return false;
172 } else {
173 futexWait(futex&: *ptr, expectedValue: curValue);
174 }
175
176 curValue = u.loadAcquire();
177 if (IsTimed)
178 remainingTime = timer.remainingTimeNSecs();
179
180 // try to acquire
181 while (futexAvailCounter(v: curValue) >= n) {
182 quintptr newValue = curValue - nn;
183 if (u.testAndSetOrdered(expectedValue: curValue, newValue, currentValue&: curValue))
184 return true; // succeeded!
185 }
186
187 // not enough tokens available, put us to wait
188 if (remainingTime == 0)
189 return false;
190 }
191}
192
193template <bool IsTimed> bool futexSemaphoreTryAcquire(QBasicAtomicInteger<quintptr> &u, int n, int timeout)
194{
195 // Try to acquire without waiting (we still loop because the testAndSet
196 // call can fail).
197 quintptr nn = unsigned(n);
198 if (futexHasWaiterCount)
199 nn |= quint64(nn) << 32; // token count replicated in high word
200
201 quintptr curValue = u.loadAcquire();
202 while (futexAvailCounter(v: curValue) >= n) {
203 // try to acquire
204 quintptr newValue = curValue - nn;
205 if (u.testAndSetOrdered(expectedValue: curValue, newValue, currentValue&: curValue))
206 return true; // succeeded!
207 }
208 if (timeout == 0)
209 return false;
210
211 // we need to wait
212 constexpr quintptr oneWaiter = quintptr(Q_UINT64_C(1) << 32); // zero on 32-bit
213 if constexpr (futexHasWaiterCount) {
214 // We don't use the fetched value from above so futexWait() fails if
215 // it changed after the testAndSetOrdered above.
216 quint32 waiterCount = (quint64(curValue) >> 32) & 0x7fffffffU;
217 if (waiterCount == 0x7fffffffU) {
218 qCritical() << "Waiter count overflow in QSemaphore";
219 return false;
220 }
221
222 // increase the waiter count
223 u.fetchAndAddRelaxed(valueToAdd: oneWaiter);
224 curValue += oneWaiter;
225
226 // Also adjust nn to subtract oneWaiter when we succeed in acquiring.
227 nn += oneWaiter;
228 }
229
230 if (futexSemaphoreTryAcquire_loop<IsTimed>(u, curValue, nn, timeout))
231 return true;
232
233 Q_ASSERT(IsTimed);
234
235 if (futexHasWaiterCount) {
236 // decrement the number of threads waiting
237 Q_ASSERT(futexHigh32(&u)->loadRelaxed() & 0x7fffffffU);
238 u.fetchAndSubRelaxed(valueToAdd: oneWaiter);
239 }
240 return false;
241}

253/*!
254 Creates a new semaphore and initializes the number of resources
255 it guards to \a n (by default, 0).
256
257 \sa release(), available()
258*/
259QSemaphore::QSemaphore(int n)
260{
261 Q_ASSERT_X(n >= 0, "QSemaphore", "parameter 'n' must be non-negative");
262 if (futexAvailable()) {
263 quintptr nn = unsigned(n);
264 if (futexHasWaiterCount)
265 nn |= quint64(nn) << 32; // token count replicated in high word
266 u.storeRelaxed(newValue: nn);
267 } else {
268 d = new QSemaphorePrivate(n);
269 }
270}

284/*!
285 Tries to acquire \c n resources guarded by the semaphore. If \a n
286 > available(), this call will block until enough resources are
287 available.
288
289 \sa release(), available(), tryAcquire()
290*/
291void QSemaphore::acquire(int n)
292{
293 Q_ASSERT_X(n >= 0, "QSemaphore::acquire", "parameter 'n' must be non-negative");
294
295 if (futexAvailable()) {
296 futexSemaphoreTryAcquire<false>(u&: u, n, timeout: -1);
297 return;
298 }
299
300 const auto sufficientResourcesAvailable = [this, n] { return d->avail >= n; };
301
302 auto locker = qt_unique_lock(mutex&: d->mutex);
303 d->cond.wait(lock&: locker, p: sufficientResourcesAvailable);
304 d->avail -= n;
305}

307/*!
308 Releases \a n resources guarded by the semaphore.
309
310 This function can be used to "create" resources as well. For
311 example:
312
313 \snippet code/src_corelib_thread_qsemaphore.cpp 1
314
315 QSemaphoreReleaser is a \l{http://en.cppreference.com/w/cpp/language/raii}{RAII}
316 wrapper around this function.
317
318 \sa acquire(), available(), QSemaphoreReleaser
319*/
320void QSemaphore::release(int n)
321{
322 Q_ASSERT_X(n >= 0, "QSemaphore::release", "parameter 'n' must be non-negative");
323
324 if (futexAvailable()) {
325 quintptr nn = unsigned(n);
326 if (futexHasWaiterCount)
327 nn |= quint64(nn) << 32; // token count replicated in high word
328 quintptr prevValue = u.loadRelaxed();
329 quintptr newValue;
330 do { // loop just to ensure the operations are done atomically
331 newValue = prevValue + nn;
332 newValue &= (futexNeedsWakeAllBit - 1);
333 } while (!u.testAndSetRelease(expectedValue: prevValue, newValue, currentValue&: prevValue));
334 if (futexNeedsWake(v: prevValue)) {
335#ifdef FUTEX_OP
336 if (futexHasWaiterCount) {
337 /*
338 On 64-bit systems, the single-token waiters wait on the low half
339 and the multi-token waiters wait on the upper half. So we ask
340 the kernel to wake up n single-token waiters and all multi-token
341 waiters (if any), and clear the multi-token wait bit.
342
343 atomic {
344 int oldval = *upper;
345 *upper = oldval | 0;
346 futexWake(lower, n);
347 if (oldval != 0) // always true
348 futexWake(upper, INT_MAX);
349 }
350 */
351 quint32 op = FUTEX_OP_OR;
352 quint32 oparg = 0;
353 quint32 cmp = FUTEX_OP_CMP_NE;
354 quint32 cmparg = 0;
355 futexWakeOp(futex1&: *futexLow32(ptr: &u), wake1: n, INT_MAX, futex2&: *futexHigh32(ptr: &u), FUTEX_OP(op, oparg, cmp, cmparg));
356 return;
357 }
358#endif
359 // Unset the bit and wake everyone. There are two possibilities
360 // under which a thread can set the bit between the AND and the
361 // futexWake:
362 // 1) it did see the new counter value, but it wasn't enough for
363 // its acquisition anyway, so it has to wait;
364 // 2) it did not see the new counter value, in which case its
365 // futexWait will fail.
366 if (futexHasWaiterCount) {
367 futexWakeAll(futex&: *futexLow32(ptr: &u));
368 futexWakeAll(futex&: *futexHigh32(ptr: &u));
369 } else {
370 futexWakeAll(futex&: u);
371 }
372 }
373 return;
374 }
375
376 const auto locker = qt_scoped_lock(mutex&: d->mutex);
377 d->avail += n;
378 d->cond.notify_all();
379}

381/*!
382 Returns the number of resources currently available to the
383 semaphore. This number can never be negative.
384
385 \sa acquire(), release()
386*/
387int QSemaphore::available() const
388{
389 if (futexAvailable())
390 return futexAvailCounter(v: u.loadRelaxed());
391
392 const auto locker = qt_scoped_lock(mutex&: d->mutex);
393 return d->avail;
394}

396/*!
397 Tries to acquire \c n resources guarded by the semaphore and
398 returns \c true on success. If available() < \a n, this call
399 immediately returns \c false without acquiring any resources.
400
401 Example:
402
403 \snippet code/src_corelib_thread_qsemaphore.cpp 2
404
405 \sa acquire()
406*/
407bool QSemaphore::tryAcquire(int n)
408{
409 Q_ASSERT_X(n >= 0, "QSemaphore::tryAcquire", "parameter 'n' must be non-negative");
410
411 if (futexAvailable())
412 return futexSemaphoreTryAcquire<false>(u&: u, n, timeout: 0);
413
414 const auto locker = qt_scoped_lock(mutex&: d->mutex);
415 if (n > d->avail)
416 return false;
417 d->avail -= n;
418 return true;
419}
420
421/*!
422 Tries to acquire \c n resources guarded by the semaphore and
423 returns \c true on success. If available() < \a n, this call will
424 wait for at most \a timeout milliseconds for resources to become
425 available.
426
427 Note: Passing a negative number as the \a timeout is equivalent to
428 calling acquire(), i.e. this function will wait forever for
429 resources to become available if \a timeout is negative.
430
431 Example:
432
433 \snippet code/src_corelib_thread_qsemaphore.cpp 3
434
435 \sa acquire()
436*/
437bool QSemaphore::tryAcquire(int n, int timeout)
438{
439 if (timeout < 0) {
440 acquire(n);
441 return true;
442 }
443
444 if (timeout == 0)
445 return tryAcquire(n);
446
447 Q_ASSERT_X(n >= 0, "QSemaphore::tryAcquire", "parameter 'n' must be non-negative");
448
449 if (futexAvailable())
450 return futexSemaphoreTryAcquire<true>(u&: u, n, timeout);
451
452 using namespace std::chrono;
453 const auto sufficientResourcesAvailable = [this, n] { return d->avail >= n; };
454
455 auto locker = qt_unique_lock(mutex&: d->mutex);
456 if (!d->cond.wait_for(lock&: locker, rtime: milliseconds{timeout}, p: sufficientResourcesAvailable))
457 return false;
458 d->avail -= n;
459 return true;
460}

507/*!
508 \class QSemaphoreReleaser
509 \brief The QSemaphoreReleaser class provides exception-safe deferral of a QSemaphore::release() call.
510 \since 5.10
511 \ingroup thread
512 \inmodule QtCore
513
514 \reentrant
515
516 QSemaphoreReleaser can be used wherever you would otherwise use
517 QSemaphore::release(). Constructing a QSemaphoreReleaser defers the
518 release() call on the semaphore until the QSemaphoreReleaser is
519 destroyed (see
520 \l{http://en.cppreference.com/w/cpp/language/raii}{RAII pattern}).
521
522 You can use this to reliably release a semaphore to avoid dead-lock
523 in the face of exceptions or early returns:
524
525 \snippet code/src_corelib_thread_qsemaphore.cpp 4
526
527 If an early return is taken or an exception is thrown before the
528 \c{sem.release()} call is reached, the semaphore is not released,
529 possibly preventing the thread waiting in the corresponding
530 \c{sem.acquire()} call from ever continuing execution.
531
532 When using RAII instead:
533
534 \snippet code/src_corelib_thread_qsemaphore.cpp 5
535
536 this can no longer happen, because the compiler will make sure that
537 the QSemaphoreReleaser destructor is always called, and therefore
538 the semaphore is always released.
539
540 QSemaphoreReleaser is move-enabled and can therefore be returned
541 from functions to transfer responsibility for releasing a semaphore
542 out of a function or a scope:
543
544 \snippet code/src_corelib_thread_qsemaphore.cpp 6
545
546 A QSemaphoreReleaser can be canceled by a call to cancel(). A canceled
547 semaphore releaser will no longer call QSemaphore::release() in its
548 destructor.
549
550 \sa QMutexLocker
551*/
