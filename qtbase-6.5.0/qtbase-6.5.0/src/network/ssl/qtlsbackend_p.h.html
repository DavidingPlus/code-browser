140// TLSTODO: consider making those into virtuals in QTlsBackend. After all, we ask the backend
141// to return those pointers if the functionality is supported, but it's a bit odd to have
142// this level of indirection. They are not parts of the classes above because ...
143// you'd then have to ask backend to create a certificate to ... call those
144// functions on a certificate.

150 QList<QSslCertificate> *caCertificates,
151 const QByteArray &passPhrase);
