40/*!
41 \class QWindow
42 \inmodule QtGui
43 \since 5.0
44 \brief The QWindow class represents a window in the underlying windowing system.
45
46 A window that is supplied a parent becomes a native child window of
47 their parent window.
48
49 An application will typically use QWidget or QQuickView for its UI, and not
50 QWindow directly. Still, it is possible to render directly to a QWindow
51 with QBackingStore or QOpenGLContext, when wanting to keep dependencies to
52 a minimum or when wanting to use OpenGL directly. The
53 \l{Raster Window Example} and \l{OpenGL Window Example}
54 are useful reference examples for how to render to a QWindow using
55 either approach.
56
57 \section1 Resource Management
58
59 Windows can potentially use a lot of memory. A usual measurement is
60 width times height times color depth. A window might also include multiple
61 buffers to support double and triple buffering, as well as depth and stencil
62 buffers. To release a window's memory resources, call the destroy() function.
63
64 \section1 Content Orientation
65
66 QWindow has reportContentOrientationChange() that can be used to specify
67 the layout of the window contents in relation to the screen. The content
68 orientation is simply a hint to the windowing system about which
69 orientation the window contents are in. It's useful when you wish to keep
70 the same window size, but rotate the contents instead, especially when
71 doing rotation animations between different orientations. The windowing
72 system might use this value to determine the layout of system popups or
73 dialogs.
74
75 \section1 Visibility and Windowing System Exposure
76
77 By default, the window is not visible, and you must call setVisible(true),
78 or show() or similar to make it visible. To make a window hidden again,
79 call setVisible(false) or hide(). The visible property describes the state
80 the application wants the window to be in. Depending on the underlying
81 system, a visible window might still not be shown on the screen. It could,
82 for instance, be covered by other opaque windows or moved outside the
83 physical area of the screen. On windowing systems that have exposure
84 notifications, the isExposed() accessor describes whether the window should
85 be treated as directly visible on screen. The exposeEvent() function is
86 called whenever an area of the window is invalidated, for example due to the
87 exposure in the windowing system changing. On windowing systems that do not
88 make this information visible to the application, isExposed() will simply
89 return the same value as isVisible().
90
91 QWindow::Visibility queried through visibility() is a convenience API
92 combining the functions of visible() and windowStates().
93
94 \section1 Rendering
95
96 There are two Qt APIs that can be used to render content into a window,
97 QBackingStore for rendering with a QPainter and flushing the contents
98 to a window with type QSurface::RasterSurface, and QOpenGLContext for
99 rendering with OpenGL to a window with type QSurface::OpenGLSurface.
100
101 The application can start rendering as soon as isExposed() returns \c true,
102 and can keep rendering until it isExposed() returns \c false. To find out when
103 isExposed() changes, reimplement exposeEvent(). The window will always get
104 a resize event before the first expose event.
105
106 \section1 Initial Geometry
107
108 If the window's width and height are left uninitialized, the window will
109 get a reasonable default geometry from the platform window. If the position
110 is left uninitialized, then the platform window will allow the windowing
111 system to position the window. For example on X11, the window manager
112 usually does some kind of smart positioning to try to avoid having new
113 windows completely obscure existing windows. However setGeometry()
114 initializes both the position and the size, so if you want a fixed size but
115 an automatic position, you should call resize() or setWidth() and
116 setHeight() instead.
117*/

229/*!
230 \enum QWindow::Visibility
231 \since 5.1
232
233 This enum describes what part of the screen the window occupies or should
234 occupy.
235
236 \value Windowed The window occupies part of the screen, but not necessarily
237 the entire screen. This state will occur only on windowing systems which
238 support showing multiple windows simultaneously. In this state it is
239 possible for the user to move and resize the window manually, if
240 WindowFlags permit it and if it is supported by the windowing system.
241
242 \value Minimized The window is reduced to an entry or icon on the task bar,
243 dock, task list or desktop, depending on how the windowing system handles
244 minimized windows.
245
246 \value Maximized The window occupies one entire screen, and the titlebar is
247 still visible. On most windowing systems this is the state achieved by
248 clicking the maximize button on the toolbar.
249
250 \value FullScreen The window occupies one entire screen, is not resizable,
251 and there is no titlebar. On some platforms which do not support showing
252 multiple simultaneous windows, this can be the usual visibility when the
253 window is not hidden.
254
255 \value AutomaticVisibility This means to give the window a default visible
256 state, which might be fullscreen or windowed depending on the platform.
257 It can be given as a parameter to setVisibility but will never be
258 read back from the visibility accessor.
259
260 \value Hidden The window is not visible in any way, however it may remember
261 a latent visibility which can be restored by setting AutomaticVisibility.
262*/

307 }

310/*
311 Subclasses may override this function to run custom setVisible
312 logic. Subclasses that do so must call the base class implementation
313 at some point to make the native window visible, and must not
314 call QWindow::setVisble() since that will recurse back here.
315*/
316void QWindowPrivate::setVisible(bool visible)
317{
318 Q_Q(QWindow);
319
320 if (this->visible != visible) {
321 this->visible = visible;
322 emit q->visibleChanged(arg: visible);
323 updateVisibility();
324 } else if (platformWindow) {
325 // Visibility hasn't changed, and the platform window is in sync
326 return;
327 }
328
329 if (!platformWindow) {
330 // If we have a parent window, but the parent hasn't been created yet, we
331 // can defer creation until the parent is created or we're re-parented.
332 if (parentWindow && !parentWindow->handle())
333 return;
334
335 // We only need to create the window if it's being shown
336 if (visible) {
337 // FIXME: At this point we've already updated the visible state of
338 // the QWindow, so if the platform layer reads the window state during
339 // creation, and reflects that in the native window, it will end up
340 // with a visible window. This may in turn result in resize or expose
341 // events from the platform before we have sent the show event below.
342 q->create();
343 }
344 }
345
346 if (visible) {
347 // remove posted quit events when showing a new window
348 QCoreApplication::removePostedEvents(qApp, eventType: QEvent::Quit);
349
350 if (q->type() == Qt::Window) {
351 QGuiApplicationPrivate *app_priv = QGuiApplicationPrivate::instance();
352 QString &firstWindowTitle = app_priv->firstWindowTitle;
353 if (!firstWindowTitle.isEmpty()) {
354 q->setTitle(firstWindowTitle);
355 firstWindowTitle = QString();
356 }
357 if (!app_priv->forcedWindowIcon.isNull())
358 q->setIcon(app_priv->forcedWindowIcon);
359
360 // Handling of the -qwindowgeometry, -geometry command line arguments
361 static bool geometryApplied = false;
362 if (!geometryApplied) {
363 geometryApplied = true;
364 QGuiApplicationPrivate::applyWindowGeometrySpecificationTo(window: q);
365 }
366 }
367
368 QShowEvent showEvent;
369 QGuiApplication::sendEvent(receiver: q, event: &showEvent);
370 }
371
372 if (q->isModal()) {
373 if (visible)
374 QGuiApplicationPrivate::showModalWindow(window: q);
375 else
376 QGuiApplicationPrivate::hideModalWindow(window: q);
377 // QShapedPixmapWindow is used on some platforms for showing a drag pixmap, so don't block
378 // input to this window as it is performing a drag - QTBUG-63846
379 } else if (visible && QGuiApplication::modalWindow()
380#if QT_CONFIG(draganddrop)
381 && !qobject_cast<QShapedPixmapWindow *>(object: q)
382#endif // QT_CONFIG(draganddrop)
383 ) {
384 QGuiApplicationPrivate::updateBlockedStatus(window: q);
385 }
386
387#ifndef QT_NO_CURSOR
388 if (visible && (hasCursor || QGuiApplication::overrideCursor()))
389 applyCursor();
390#endif
391
392 if (platformWindow)
393 platformWindow->setVisible(visible);
394
395 if (!visible) {
396 QHideEvent hideEvent;
397 QGuiApplication::sendEvent(receiver: q, event: &hideEvent);
398 }
399}

401void QWindowPrivate::updateVisibility()
402{
403 Q_Q(QWindow);
404
405 QWindow::Visibility old = visibility;
406
407 if (!visible)
408 visibility = QWindow::Hidden;
409 else if (windowState & Qt::WindowMinimized)
410 visibility = QWindow::Minimized;
411 else if (windowState & Qt::WindowFullScreen)
412 visibility = QWindow::FullScreen;
413 else if (windowState & Qt::WindowMaximized)
414 visibility = QWindow::Maximized;
415 else
416 visibility = QWindow::Windowed;
417
418 if (visibility != old)
419 emit q->visibilityChanged(visibility);
420}

422void QWindowPrivate::updateSiblingPosition(SiblingPosition position)
423{
424 Q_Q(QWindow);
425
426 if (!q->parent())
427 return;
428
429 QObjectList &siblings = q->parent()->d_ptr->children;
430
431 const int siblingCount = siblings.size() - 1;
432 if (siblingCount == 0)
433 return;
434
435 const int currentPosition = siblings.indexOf(t: q);
436 Q_ASSERT(currentPosition >= 0);
437
438 const int targetPosition = position == PositionTop ? siblingCount : 0;
439
440 if (currentPosition == targetPosition)
441 return;
442
443 siblings.move(from: currentPosition, to: targetPosition);
444}

476void QWindowPrivate::setTopLevelScreen(QScreen *newScreen, bool recreate)
477{
478 Q_Q(QWindow);
479 if (parentWindow) {
480 qWarning() << q << '(' << newScreen << "): Attempt to set a screen on a child window.";
481 return;
482 }
483 if (newScreen != topLevelScreen) {
484 const bool shouldRecreate = recreate && windowRecreationRequired(newScreen);
485 const bool shouldShow = visibilityOnDestroy && !topLevelScreen;
486 if (shouldRecreate && platformWindow)
487 q->destroy();
488 connectToScreen(screen: newScreen);
489 if (shouldShow)
490 q->setVisible(true);
491 else if (newScreen && shouldRecreate)
492 create(recursive: true);
493 emitScreenChangedRecursion(newScreen);
494 }
495}

497void QWindowPrivate::create(bool recursive, WId nativeHandle)
498{
499 Q_Q(QWindow);
500 if (platformWindow)
501 return;
502
503 // avoid losing update requests when re-creating
504 const bool needsUpdate = updateRequestPending;
505 // the platformWindow, if there was one, is now gone, so make this flag reflect reality now
506 updateRequestPending = false;
507
508 if (q->parent())
509 q->parent()->create();
510
511 // QPlatformWindow will poll geometry() during construction below. Set the
512 // screen here so that high-dpi scaling will use the correct scale factor.
513 if (q->isTopLevel()) {
514 if (QScreen *screen = screenForGeometry(rect: geometry))
515 setTopLevelScreen(newScreen: screen, recreate: false);
516 }
517
518 QPlatformIntegration *platformIntegration = QGuiApplicationPrivate::platformIntegration();
519 platformWindow = nativeHandle ? platformIntegration->createForeignWindow(q, nativeHandle)
520 : platformIntegration->createPlatformWindow(window: q);
521 Q_ASSERT(platformWindow);
522
523 if (!platformWindow) {
524 qWarning() << "Failed to create platform window for" << q << "with flags" << q->flags();
525 return;
526 }
527
528 platformWindow->initialize();
529
530 QObjectList childObjects = q->children();
531 for (int i = 0; i < childObjects.size(); i ++) {
532 QObject *object = childObjects.at(i);
533 if (!object->isWindowType())
534 continue;
535
536 QWindow *childWindow = static_cast<QWindow *>(object);
537 if (recursive)
538 childWindow->d_func()->create(recursive);
539
540 // The child may have had deferred creation due to this window not being created
541 // at the time setVisible was called, so we re-apply the visible state, which
542 // may result in creating the child, and emitting the appropriate signals.
543 if (childWindow->isVisible())
544 childWindow->setVisible(true);
545
546 if (QPlatformWindow *childPlatformWindow = childWindow->d_func()->platformWindow)
547 childPlatformWindow->setParent(this->platformWindow);
548 }
549
550 QPlatformSurfaceEvent e(QPlatformSurfaceEvent::SurfaceCreated);
551 QGuiApplication::sendEvent(receiver: q, event: &e);
552
553 if (needsUpdate)
554 q->requestUpdate();
555}

570void QWindowPrivate::setMinOrMaxSize(QSize *oldSizeMember, const QSize &size,
571 qxp::function_ref<void()> funcWidthChanged,
572 qxp::function_ref<void()> funcHeightChanged)
573{
574 Q_Q(QWindow);
575 Q_ASSERT(oldSizeMember);
576 const QSize adjustedSize =
577 size.expandedTo(otherSize: QSize(0, 0)).boundedTo(otherSize: QSize(QWINDOWSIZE_MAX, QWINDOWSIZE_MAX));
578 if (*oldSizeMember == adjustedSize)
579 return;
580 const bool widthChanged = adjustedSize.width() != oldSizeMember->width();
581 const bool heightChanged = adjustedSize.height() != oldSizeMember->height();
582 *oldSizeMember = adjustedSize;
583
584 if (platformWindow && q->isTopLevel())
585 platformWindow->propagateSizeHints();
586
587 if (widthChanged)
588 funcWidthChanged();
589 if (heightChanged)
590 funcHeightChanged();
591
592 // resize window if current size is outside of min and max limits
593 if (minimumSize.width() <= maximumSize.width()
594 || minimumSize.height() <= maximumSize.height()) {
595 q->resize(newSize: q->geometry().size().expandedTo(otherSize: minimumSize).boundedTo(otherSize: maximumSize));
596 }
597}

715/*!
716 Sets the \a parent Window. This will lead to the windowing system managing
717 the clip of the window, so it will be clipped to the \a parent window.
718
719 Setting \a parent to be \nullptr will make the window become a top level
720 window.
721
722 If \a parent is a window created by fromWinId(), then the current window
723 will be embedded inside \a parent, if the platform supports it.
724*/
725void QWindow::setParent(QWindow *parent)
726{
727 parent = nonDesktopParent(parent);
728
729 Q_D(QWindow);
730 if (d->parentWindow == parent)
731 return;
732
733 QScreen *newScreen = parent ? parent->screen() : screen();
734 if (d->windowRecreationRequired(newScreen)) {
735 qWarning() << this << '(' << parent << "): Cannot change screens (" << screen() << newScreen << ')';
736 return;
737 }
738
739 QObject::setParent(parent);
740 d->parentWindow = parent;
741
742 if (parent)
743 d->disconnectFromScreen();
744 else
745 d->connectToScreen(screen: newScreen);
746
747 // If we were set visible, but not created because we were a child, and we're now
748 // re-parented into a created parent, or to being a top level, we need re-apply the
749 // visibility state, which will also create.
750 if (isVisible() && (!parent || parent->handle()))
751 setVisible(true);
752
753 if (d->platformWindow) {
754 if (parent)
755 parent->create();
756
757 d->platformWindow->setParent(parent ? parent->d_func()->platformWindow : nullptr);
758 }
759
760 QGuiApplicationPrivate::updateBlockedStatus(window: this);
761}

1985 // a reimplemented event() will not get called in the subclasses at that

1995 // this window during destruction (e.g. as a result of platform events).
1996 delete std::exchange(obj&: platformWindow, new_val: nullptr);
1997
1998 if (QGuiApplicationPrivate::focus_window == q)

2000 if (QGuiApplicationPrivate::currentMouseWindow == q)
