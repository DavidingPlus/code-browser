31/*!
32 \class QAccessible
33 \brief The QAccessible class provides enums and static functions
34 related to accessibility.
35
36 \ingroup accessibility
37 \inmodule QtGui
38
39 This class is part of \l {Accessibility for QWidget Applications}.
40
41 Accessible applications can be used by people who are not able to
42 use applications by conventional means.
43
44 The functions in this class are used for communication between
45 accessible applications (also called AT Servers) and
46 accessibility tools (AT Clients), such as screen readers and
47 braille displays. Clients and servers communicate in the following way:
48
49 \list
50 \li \e{AT Servers} notify the clients about events through calls to the
51 updateAccessibility() function.
52
53 \li \e{AT Clients} request information about the objects in the server.
54 The QAccessibleInterface class is the core interface, and encapsulates
55 this information in a pure virtual API. Implementations of the interface
56 are provided by Qt through the queryAccessibleInterface() API.
57 \endlist
58
59 The communication between servers and clients is initialized by
60 the setRootObject() function. Function pointers can be installed
61 to replace or extend the default behavior of the static functions
62 in QAccessible.
63
64 Qt supports Microsoft Active Accessibility (MSAA), \macos
65 Accessibility, and the Unix/X11 AT-SPI standard. Other backends
66 can be supported using QAccessibleBridge.
67
68 In the Unix/X11 AT-SPI implementation, applications become accessible
69 when two conditions are met:
70 \list
71 \li org.a11y.Status.IsEnabled DBus property is true
72 \li org.a11y.Status.ScreenReaderEnabled DBus property is true
73 \endlist
74 An alternative to setting the DBus AT-SPI properties is to set
75 the QT_LINUX_ACCESSIBILITY_ALWAYS_ON environment variable.
76
77 In addition to QAccessible's static functions, Qt offers one
78 generic interface, QAccessibleInterface, that can be used to wrap
79 all widgets and objects (e.g., QPushButton). This single
80 interface provides all the metadata necessary for the assistive
81 technologies. Qt provides implementations of this interface for
82 its built-in widgets as plugins.
83
84 When you develop custom widgets, you can create custom subclasses
85 of QAccessibleInterface and distribute them as plugins (using
86 QAccessiblePlugin) or compile them into the application.
87 Likewise, Qt's predefined accessibility support can be built as
88 plugin (the default) or directly into the Qt library. The main
89 advantage of using plugins is that the accessibility classes are
90 only loaded into memory if they are actually used; they don't
91 slow down the common case where no assistive technology is being
92 used.
93
94 Qt also includes two convenience classes, QAccessibleObject and
95 QAccessibleWidget, that inherit from QAccessibleInterface and
96 provide the lowest common denominator of metadata (e.g., widget
97 geometry, window title, basic help text). You can use them as
98 base classes when wrapping your custom QObject or QWidget
99 subclasses.
100
101 \sa QAccessibleInterface
102*/

622/*!
623 If a QAccessibleInterface implementation exists for the given \a object,
624 this function returns a pointer to the implementation; otherwise it
625 returns \nullptr.
626
627 The function calls all installed factory functions (from most
628 recently installed to least recently installed) until one is found
629 that provides an interface for the class of \a object. If no
630 factory can provide an accessibility implementation for the class
631 the function loads installed accessibility plugins, and tests if
632 any of the plugins can provide the implementation.
633
634 If no implementation for the object's class is available, the
635 function tries to find an implementation for the object's parent
636 class, using the above strategy.
637
638 All interfaces are managed by an internal cache and should not be deleted.
639*/
640QAccessibleInterface *QAccessible::queryAccessibleInterface(QObject *object)
641{
642 if (!object)
643 return nullptr;
644
645 if (Id id = QAccessibleCache::instance()->idForObject(obj: object))
646 return QAccessibleCache::instance()->interfaceForId(id);
647
648 // Create a QAccessibleInterface for the object class. Start by the most
649 // derived class and walk up the class hierarchy.
650 const QMetaObject *mo = object->metaObject();
651 const auto *objectPriv = QObjectPrivate::get(o: object);
652 /*
653 We do not want to cache each and every QML metaobject (Button_QMLTYPE_124,
654 Button_QMLTYPE_125, etc.). Those dynamic metaobjects shouldn't have an
655 accessible interface in any case. Instead, we start the whole checking
656 with the first non-dynamic meta-object. To avoid potential regressions
657 in other areas of Qt that also use dynamic metaobjects, we only do this
658 for objects that are QML-related (approximated by checking whether they
659 have ddata set).
660 */
661 const bool qmlRelated = !objectPriv->isDeletingChildren &&
662 objectPriv->declarativeData;
663 while (qmlRelated && mo) {
664 auto mop = QMetaObjectPrivate::get(metaobject: mo);
665 if (!mop || !(mop->flags & DynamicMetaObject))
666 break;
667
668 mo = mo->superClass();
669 };
670 while (mo) {
671 const QString cn = QLatin1StringView(mo->className());
672
673 // Check if the class has a InterfaceFactory installed.
674 for (int i = qAccessibleFactories()->size(); i > 0; --i) {
675 InterfaceFactory factory = qAccessibleFactories()->at(i: i - 1);
676 if (QAccessibleInterface *iface = factory(cn, object)) {
677 QAccessibleCache::instance()->insert(object, iface);
678 Q_ASSERT(QAccessibleCache::instance()->containsObject(object));
679 return iface;
680 }
681 }
682 // Find a QAccessiblePlugin (factory) for the class name. If there's
683 // no entry in the cache try to create it using the plugin loader.
684 if (!qAccessiblePlugins()->contains(key: cn)) {
685 QAccessiblePlugin *factory = nullptr; // 0 means "no plugin found". This is cached as well.
686 const int index = acLoader()->indexOf(needle: cn);
687 if (index != -1)
688 factory = qobject_cast<QAccessiblePlugin *>(object: acLoader()->instance(index));
689 qAccessiblePlugins()->insert(key: cn, value: factory);
690 }
691
692 // At this point the cache should contain a valid factory pointer or 0:
693 Q_ASSERT(qAccessiblePlugins()->contains(cn));
694 QAccessiblePlugin *factory = qAccessiblePlugins()->value(key: cn);
695 if (factory) {
696 QAccessibleInterface *result = factory->create(key: cn, object);
697 if (result) {
698 QAccessibleCache::instance()->insert(object, iface: result);
699 Q_ASSERT(QAccessibleCache::instance()->containsObject(object));
700 }
701 return result;
702 }
703 mo = mo->superClass();
704 }
705
706 if (object == qApp) {
707 QAccessibleInterface *appInterface = new QAccessibleApplication;
708 QAccessibleCache::instance()->insert(object, iface: appInterface);
709 Q_ASSERT(QAccessibleCache::instance()->containsObject(qApp));
710 return appInterface;
711 }
712
713 return nullptr;
714}

865/*!
866 \internal
867 \brief getBoundaries is a helper function to find the accessible text boundaries for QTextCursor based documents.
868 \param documentCursor a valid cursor bound to the document (not null). It needs to ba at the position to look for the boundary
869 \param boundaryType the type of boundary to find
870 \return the boundaries as pair
871*/
872QPair< int, int > QAccessible::qAccessibleTextBoundaryHelper(const QTextCursor &offsetCursor, TextBoundaryType boundaryType)
873{
874 Q_ASSERT(!offsetCursor.isNull());
875
876 QTextCursor endCursor = offsetCursor;
877 endCursor.movePosition(op: QTextCursor::End);
878 int characterCount = endCursor.position();
879
880 QPair<int, int> result;
881 QTextCursor cursor = offsetCursor;
882 switch (boundaryType) {
883 case CharBoundary:
884 result.first = cursor.position();
885 cursor.movePosition(op: QTextCursor::NextCharacter, QTextCursor::KeepAnchor);
886 result.second = cursor.position();
887 break;
888 case WordBoundary:
889 cursor.movePosition(op: QTextCursor::StartOfWord, QTextCursor::MoveAnchor);
890 result.first = cursor.position();
891 cursor.movePosition(op: QTextCursor::EndOfWord, QTextCursor::KeepAnchor);
892 result.second = cursor.position();
893 break;
894 case SentenceBoundary: {
895 // QCursor does not provide functionality to move to next sentence.
896 // We therefore find the current block, then go through the block using
897 // QTextBoundaryFinder and find the sentence the \offset represents
898 cursor.movePosition(op: QTextCursor::StartOfBlock, QTextCursor::MoveAnchor);
899 result.first = cursor.position();
900 cursor.movePosition(op: QTextCursor::EndOfBlock, QTextCursor::KeepAnchor);
901 result.second = cursor.position();
902 QString blockText = cursor.selectedText();
903 const int offsetWithinBlockText = offsetCursor.position() - result.first;
904 QTextBoundaryFinder sentenceFinder(QTextBoundaryFinder::Sentence, blockText);
905 sentenceFinder.setPosition(offsetWithinBlockText);
906 int prevBoundary = offsetWithinBlockText;
907 int nextBoundary = offsetWithinBlockText;
908 if (!(sentenceFinder.boundaryReasons() & QTextBoundaryFinder::StartOfItem))
909 prevBoundary = sentenceFinder.toPreviousBoundary();
910 nextBoundary = sentenceFinder.toNextBoundary();
911 if (nextBoundary != -1)
912 result.second = result.first + nextBoundary;
913 if (prevBoundary != -1)
914 result.first += prevBoundary;
915 break; }
916 case LineBoundary:
917 cursor.movePosition(op: QTextCursor::StartOfLine, QTextCursor::MoveAnchor);
918 result.first = cursor.position();
919 cursor.movePosition(op: QTextCursor::EndOfLine, QTextCursor::KeepAnchor);
920 result.second = cursor.position();
921 break;
922 case ParagraphBoundary:
923 cursor.movePosition(op: QTextCursor::StartOfBlock, QTextCursor::MoveAnchor);
924 result.first = cursor.position();
925 cursor.movePosition(op: QTextCursor::EndOfBlock, QTextCursor::KeepAnchor);
926 result.second = cursor.position();
927 break;
928 case NoBoundary:
929 result.first = 0;
930 result.second = characterCount;
931 break;
932 }
933 return result;
934}

936/*!
937 \class QAccessibleInterface
938 \brief The QAccessibleInterface class defines an interface that exposes information
939 about accessible objects.
940
941 \ingroup accessibility
942 \inmodule QtGui
943
944 This class is part of \l {Accessibility for QWidget Applications}.
945
946 Accessibility tools (also called AT Clients), such as screen readers
947 or braille displays, require high-level information about
948 accessible objects in an application. Accessible objects provide
949 specialized input and output methods, making it possible for users
950 to use accessibility tools with enabled applications (AT Servers).
951
952 Every element that the user needs to interact with or react to is
953 an accessible object, and should provide this information. These
954 are mainly visual objects, such as widgets and widget elements, but
955 can also be content, such as sounds.
956
957 The AT client uses three basic concepts to acquire information
958 about any accessible object in an application:
959 \list
960 \li \e Properties The client can read information about
961 accessible objects. In some cases the client can also modify these
962 properties; such as text in a line edit.
963 \li \e Actions The client can invoke actions like pressing a button
964 or .
965 \li \e{Relationships and Navigation} The client can traverse from one
966 accessible object to another, using the relationships between objects.
967 \endlist
968
969 The QAccessibleInterface defines the API for these three concepts.
970
971 \section1 Relationships and Navigation
972
973 The functions childCount() and indexOfChild() return the number of
974 children of an accessible object and the index a child object has
975 in its parent. The childAt() function returns a child QAccessibleInterface
976 that is found at a position. The child does not have to be a direct
977 child. This allows bypassing intermediate layers when the parent already knows the
978 top-most child. childAt() is used for hit testing (finding the object
979 under the mouse).
980
981 The relations() function provides information about the relations an
982 object has to other objects, and parent() and child() allows
983 traversing from one object to another object.
984
985 \section1 Properties
986
987 The central property of an accessible objects is what role() it
988 has. Different objects can have the same role, e.g. both the "Add
989 line" element in a scroll bar and the \c OK button in a dialog have
990 the same role, "button". The role implies what kind of
991 interaction the user can perform with the user interface element.
992
993 An object's state() property is a combination of different state
994 flags and can describe both how the object's state differs from a
995 "normal" state, e.g. it might be unavailable, and also how it
996 behaves, e.g. it might be selectable.
997
998 The text() property provides textual information about the object.
999 An object usually has a name, but can provide extended information
1000 such as a description, help text, or information about any
1001 keyboard accelerators it provides. Some objects allow changing the
1002 text() property through the setText() function, but this
1003 information is in most cases read-only.
1004
1005 The rect() property provides information about the geometry of an
1006 accessible object. This information is usually only available for
1007 visual objects.
1008
1009 \section1 Interfaces
1010
1011 To enable the user to interact with an accessible object the
1012 object must implement QAccessibleActionInterface in addition to
1013 QAccessibleInterface.
1014 Objects that support selections can define actions to change the selection.
1015
1016 There are several other interfaces that should be implemented as required.
1017 QAccessibleTextInterface should be used for bigger texts edits such as document views.
1018 This interface should not be implemented for labels/single line edits.

1021 should be implemented.

2008/*!
2009 \internal
2010 Helper for finding line breaks in textBeforeOffset/textAtOffset/textAfterOffset.
2011 \a beforeAtAfter is the line we look for. -1 for before, 0 for at and 1 for after.
2012*/
2013static QString textLineBoundary(int beforeAtAfter, const QString &text, int offset, int *startOffset, int *endOffset)
2014{
2015 Q_ASSERT(beforeAtAfter >= -1 && beforeAtAfter <= 1);
2016 Q_ASSERT(*startOffset == -1 && *endOffset == -1);
2017 int length = text.size();
2018 Q_ASSERT(offset >= 0 && offset <= length);
2019
2020 // move offset into the right range (if asking for line before or after
2021 if (beforeAtAfter == 1) {
2022 offset = text.indexOf(c: QChar::LineFeed, from: qMin(a: offset, b: length - 1));
2023 if (offset < 0)
2024 return QString(); // after the last line comes nothing
2025 ++offset; // move after the newline
2026 } else if (beforeAtAfter == -1) {
2027 offset = text.lastIndexOf(c: QChar::LineFeed, from: qMax(a: offset - 1, b: 0));
2028 if (offset < 0)
2029 return QString(); // before first line comes nothing
2030 }
2031
2032 if (offset > 0)
2033 *startOffset = text.lastIndexOf(c: QChar::LineFeed, from: offset - 1);
2034 ++*startOffset; // move to the char after the newline (0 if lastIndexOf returned -1)
2035
2036 *endOffset = text.indexOf(c: QChar::LineFeed, from: qMin(a: offset, b: length - 1)) + 1; // include newline char
2037 if (*endOffset <= 0 || *endOffset > length)
2038 *endOffset = length; // if the text doesn't end with a newline it ends at length
2039
2040 return text.mid(position: *startOffset, n: *endOffset - *startOffset);
2041}

2043/*!
2044 Returns the text item of type \a boundaryType that is close to offset \a offset
2045 and sets \a startOffset and \a endOffset values to the start and end positions
2046 of that item; returns an empty string if there is no such an item.
2047 Sets \a startOffset and \a endOffset values to -1 on error.
2048
2049 This default implementation is provided for small text edits. A word processor or
2050 text editor should provide their own efficient implementations. This function makes no
2051 distinction between paragraphs and lines.
2052
2053 \note this function can not take the cursor position into account. By convention
2054 an \a offset of -2 means that this function should use the cursor position as offset.
2055 Thus an offset of -2 must be converted to the cursor position before calling this
2056 function.
2057 An offset of -1 is used for the text length and custom implementations of this function
2058 have to return the result as if the length was passed in as offset.
2059*/
2060QString QAccessibleTextInterface::textBeforeOffset(int offset, QAccessible::TextBoundaryType boundaryType,
2061 int *startOffset, int *endOffset) const
2062{
2063 const QString txt = text(startOffset: 0, endOffset: characterCount());
2064
2065 if (offset == -1)
2066 offset = txt.size();
2067
2068 *startOffset = *endOffset = -1;
2069 if (txt.isEmpty() || offset <= 0 || offset > txt.size())
2070 return QString();
2071
2072 // type initialized just to silence a compiler warning [-Werror=maybe-uninitialized]
2073 QTextBoundaryFinder::BoundaryType type = QTextBoundaryFinder::Grapheme;
2074 switch (boundaryType) {
2075 case QAccessible::CharBoundary:
2076 type = QTextBoundaryFinder::Grapheme;
2077 break;
2078 case QAccessible::WordBoundary:
2079 type = QTextBoundaryFinder::Word;
