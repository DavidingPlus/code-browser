10/*!
11 \class QShader
12 \internal
13 \inmodule QtGui
14
15 \brief Contains multiple versions of a shader translated to multiple shading languages,
16 together with reflection metadata.
17
18 QShader is the entry point to shader code in the graphics API agnostic
19 Qt world. Instead of using GLSL shader sources, as was the custom with Qt
20 5.x, new graphics systems with backends for multiple graphics APIs, such
21 as, Vulkan, Metal, Direct3D, and OpenGL, take QShader as their input
22 whenever a shader needs to be specified.
23
24 A QShader instance is empty and thus invalid by default. To get a useful
25 instance, the two typical methods are:
26
27 \list
28
29 \li Generate the contents offline, during build time or earlier, using the
30 \c qsb command line tool. The result is a binary file that is shipped with
31 the application, read via QIODevice::readAll(), and then deserialized via
32 fromSerialized(). For more information, see QShaderBaker.
33
34 \li Generate at run time via QShaderBaker. This is an expensive operation,
35 but allows applications to use user-provided or dynamically generated
36 shader source strings.
37
38 \endlist
39
40 When used together with the Qt Rendering Hardware Interface and its
41 classes, like QRhiGraphicsPipeline, no further action is needed from the
42 application's side as these classes are prepared to consume a QShader
43 whenever a shader needs to be specified for a given stage of the graphics
44 pipeline.
45
46 Alternatively, applications can access
47
48 \list
49
50 \li the source or byte code for any of the shading language versions that
51 are included in the QShader,
52
53 \li the name of the entry point for the shader,
54
55 \li the reflection metadata containing a description of the shader's
56 inputs, outputs and resources like uniform blocks. This is essential when
57 an application or framework needs to discover the inputs of a shader at
58 runtime due to not having advance knowledge of the vertex attributes or the
59 layout of the uniform buffers used by the shader.
60
61 \endlist
62
63 QShader makes no assumption about the shading language that was used
64 as the source for generating the various versions and variants that are
65 included in it.
66
67 QShader uses implicit sharing similarly to many core Qt types, and so
68 can be returned or passed by value. Detach happens implicitly when calling
69 a setter.
70
71 For reference, QRhi expects that a QShader suitable for all its
72 backends contains at least the following:
73
74 \list
75
76 \li SPIR-V 1.0 bytecode suitable for Vulkan 1.0 or newer
77
78 \li GLSL/ES 100 source code suitable for OpenGL ES 2.0 or newer
79
80 \li GLSL 120 source code suitable for OpenGL 2.1
81
82 \li HLSL Shader Model 5.0 source code or the corresponding DXBC bytecode suitable for Direct3D 11
83
84 \li Metal Shading Language 1.2 source code or the corresponding bytecode suitable for Metal
85
86 \endlist
87
88 \sa QShaderBaker
89 */
90

103/*!
104 \class QShaderVersion
105 \internal
106 \inmodule QtGui
107
108 \brief Specifies the shading language version.
109
110 While languages like SPIR-V or the Metal Shading Language use traditional
111 version numbers, shaders for other APIs can use slightly different
112 versioning schemes. All those are mapped to a single version number in
113 here, however. For HLSL, the version refers to the Shader Model version,
114 like 5.0, 5.1, or 6.0. For GLSL an additional flag is needed to choose
115 between GLSL and GLSL/ES.
116
117 Below is a list with the most common examples of shader versions for
118 different graphics APIs:
119
120 \list
121
122 \li Vulkan (SPIR-V): 100
123 \li OpenGL: 120, 330, 440, etc.
124 \li OpenGL ES: 100 with GlslEs, 300 with GlslEs, etc.
125 \li Direct3D: 50, 51, 60
126 \li Metal: 12, 20
127 \endlist
128
129 A default constructed QShaderVersion contains a version of 100 and no
130 flags set.
131 */
132
133/*!
134 \enum QShaderVersion::Flag
135
136 Describes the flags that can be set.
137
138 \value GlslEs Indicates that GLSL/ES is meant in combination with GlslShader
139 */

141/*!
142 \class QShaderKey
143 \internal
144 \inmodule QtGui
145
146 \brief Specifies the shading language, the version with flags, and the variant.
147
148 A default constructed QShaderKey has source set to SpirvShader and
149 sourceVersion set to 100. sourceVariant defaults to StandardShader.
150 */
151
152/*!
153 \enum QShader::Source
154 Describes what kind of shader code an entry contains.
155
156 \value SpirvShader SPIR-V
157 \value GlslShader GLSL
158 \value HlslShader HLSL
159 \value DxbcShader Direct3D bytecode (HLSL compiled by \c fxc)
160 \value MslShader Metal Shading Language
161 \value DxilShader Direct3D bytecode (HLSL compiled by \c dxc)
162 \value MetalLibShader Pre-compiled Metal bytecode
163 \value WgslShader WGSL
164 */
165
166/*!
167 \enum QShader::Variant
168 Describes what kind of shader code an entry contains.
169
170 \value StandardShader The normal, unmodified version of the shader code.
171
172 \value BatchableVertexShader Vertex shader rewritten to be suitable for Qt Quick scenegraph batching.
173
174 \value UInt16IndexedVertexAsComputeShader A vertex shader meant to be used
175 in a Metal pipeline with tessellation in combination with indexed draw
176 calls sourcing index data from a uint16 index buffer. To support the Metal
177 tessellation pipeline, the vertex shader is translated to a compute shader
178 that may be dependent on the index buffer usage in the draw calls (e.g. if
179 the shader is using gl_VertexIndex), hence the need for three dedicated
180 variants.
181
182 \value UInt32IndexedVertexAsComputeShader A vertex shader meant to be used
183 in a Metal pipeline with tessellation in combination with indexed draw
184 calls sourcing index data from a uint32 index buffer. To support the Metal
185 tessellation pipeline, the vertex shader is translated to a compute shader
186 that may be dependent on the index buffer usage in the draw calls (e.g. if
187 the shader is using gl_VertexIndex), hence the need for three dedicated
188 variants.
189
190 \value NonIndexedVertexAsComputeShader A vertex shader meant to be used in
191 a Metal pipeline with tessellation in combination with non-indexed draw
192 calls. To support the Metal tessellation pipeline, the vertex shader is
193 translated to a compute shader that may be dependent on the index buffer
194 usage in the draw calls (e.g. if the shader is using gl_VertexIndex), hence
195 the need for three dedicated variants.
196 */

242{
243 if (d) {
244 if (other.d) {
245 qAtomicAssign(d, x: other.d);
246 } else {
247 if (!d->ref.deref())
248 delete d;
249 d = nullptr;
250 }
251 } else if (other.d) {
252 other.d->ref.ref();
253 d = other.d;
254 }
255 return *this;
256}

339/*!
340 Removes the source or binary shader code for a given \a key.
341 Does nothing when not found.
342 */
343void QShader::removeShader(const QShaderKey &key)
344{
345 if (!d)
346 return;
347
348 auto it = d->shaders.find(key);
349 if (it == d->shaders.end())
350 return;
351
352 detach();
353 d->shaders.erase(it);
354}

356static void writeShaderKey(QDataStream *ds, const QShaderKey &k)
357{
358 *ds << int(k.source());
359 *ds << k.sourceVersion().version();
360 *ds << k.sourceVersion().flags();
361 *ds << int(k.sourceVariant());
362}

364/*!
365 \return a serialized binary version of all the data held by the
366 QShader, suitable for writing to files or other I/O devices.
367
368 By default the latest serialization format is used. Use \a version
369 parameter to serialize for a compatibility Qt version.
370
371 \sa fromSerialized()
372 */
373QByteArray QShader::serialized(SerializedFormatVersion version) const
374{
375 static QShaderPrivate sd;
376 QShaderPrivate *dd = d ? d : &sd;
377
378 QBuffer buf;
379 QDataStream ds(&buf);
380 ds.setVersion(QDataStream::Qt_5_10);
381 if (!buf.open(openMode: QIODevice::WriteOnly))
382 return QByteArray();
383
384 const int qsbVersion = QShaderPrivate::qtQsbVersion(qtVersion: version);
385 ds << qsbVersion;
386
387 ds << int(dd->stage);
388 dd->desc.serialize(stream: &ds, version: qsbVersion);
389 ds << int(dd->shaders.size());
390 for (auto it = dd->shaders.cbegin(), itEnd = dd->shaders.cend(); it != itEnd; ++it) {
391 const QShaderKey &k(it.key());
392 writeShaderKey(ds: &ds, k);
393 const QShaderCode &shader(dd->shaders.value(key: k));
394 ds << shader.shader();
395 ds << shader.entryPoint();
396 }
397 ds << int(dd->bindings.size());
398 for (auto it = dd->bindings.cbegin(), itEnd = dd->bindings.cend(); it != itEnd; ++it) {
399 const QShaderKey &k(it.key());
400 writeShaderKey(ds: &ds, k);
401 const NativeResourceBindingMap &map(it.value());
402 ds << int(map.size());
403 for (auto mapIt = map.cbegin(), mapItEnd = map.cend(); mapIt != mapItEnd; ++mapIt) {
404 ds << mapIt.key();
405 ds << mapIt.value().first;
406 ds << mapIt.value().second;
407 }
408 }
409 ds << int(dd->combinedImageMap.size());
410 for (auto it = dd->combinedImageMap.cbegin(), itEnd = dd->combinedImageMap.cend(); it != itEnd; ++it) {
411 const QShaderKey &k(it.key());
412 writeShaderKey(ds: &ds, k);
413 const SeparateToCombinedImageSamplerMappingList &list(it.value());
414 ds << int(list.size());
415 for (auto listIt = list.cbegin(), listItEnd = list.cend(); listIt != listItEnd; ++listIt) {
416 ds << listIt->combinedSamplerName;
417 ds << listIt->textureBinding;
418 ds << listIt->samplerBinding;
419 }
420 }
421 if (qsbVersion > QShaderPrivate::QSB_VERSION_WITHOUT_NATIVE_SHADER_INFO) {
422 ds << int(dd->nativeShaderInfoMap.size());
423 for (auto it = dd->nativeShaderInfoMap.cbegin(), itEnd = dd->nativeShaderInfoMap.cend(); it != itEnd; ++it) {
424 const QShaderKey &k(it.key());
425 writeShaderKey(ds: &ds, k);
426 ds << it->flags;
427 ds << int(it->extraBufferBindings.size());
428 for (auto mapIt = it->extraBufferBindings.cbegin(), mapItEnd = it->extraBufferBindings.cend();
429 mapIt != mapItEnd; ++mapIt)
430 {
431 ds << mapIt.key();
432 ds << mapIt.value();
433 }
434 }
435 }
436
437 return qCompress(data: buf.buffer());
438}

440static void readShaderKey(QDataStream *ds, QShaderKey *k)
441{
442 int intVal;
443 *ds >> intVal;
444 k->setSource(QShader::Source(intVal));
445 QShaderVersion ver;
446 *ds >> intVal;
447 ver.setVersion(intVal);
448 *ds >> intVal;
449 ver.setFlags(QShaderVersion::Flags(intVal));
450 k->setSourceVersion(ver);
451 *ds >> intVal;
452 k->setSourceVariant(QShader::Variant(intVal));
453}

455/*!
456 Creates a new QShader instance from the given \a data.
457
458 \sa serialized()
459 */
460QShader QShader::fromSerialized(const QByteArray &data)
461{
462 QByteArray udata = qUncompress(data);
463 QBuffer buf(&udata);
464 QDataStream ds(&buf);
465 ds.setVersion(QDataStream::Qt_5_10);
466 if (!buf.open(openMode: QIODevice::ReadOnly))
467 return QShader();
468
469 QShader bs;
470 bs.detach(); // to get d created
471 QShaderPrivate *d = QShaderPrivate::get(s: &bs);
472 Q_ASSERT(d->ref.loadRelaxed() == 1); // must be detached
473 int intVal;
474 ds >> intVal;
475 d->qsbVersion = intVal;
476 if (d->qsbVersion != QShaderPrivate::QSB_VERSION
477 && d->qsbVersion != QShaderPrivate::QSB_VERSION_WITHOUT_EXTENDED_STORAGE_BUFFER_INFO
478 && d->qsbVersion != QShaderPrivate::QSB_VERSION_WITHOUT_NATIVE_SHADER_INFO
479 && d->qsbVersion != QShaderPrivate::QSB_VERSION_WITHOUT_SEPARATE_IMAGES_AND_SAMPLERS
480 && d->qsbVersion != QShaderPrivate::QSB_VERSION_WITHOUT_VAR_ARRAYDIMS
481 && d->qsbVersion != QShaderPrivate::QSB_VERSION_WITH_CBOR
482 && d->qsbVersion != QShaderPrivate::QSB_VERSION_WITH_BINARY_JSON
483 && d->qsbVersion != QShaderPrivate::QSB_VERSION_WITHOUT_BINDINGS)
484 {
485 qWarning(msg: "Attempted to deserialize QShader with unknown version %d.", d->qsbVersion);
486 return QShader();
487 }
488
489 ds >> intVal;
490 d->stage = Stage(intVal);
491 if (d->qsbVersion > QShaderPrivate::QSB_VERSION_WITH_CBOR) {
492 d->desc = QShaderDescription::deserialize(stream: &ds, version: d->qsbVersion);
493 } else if (d->qsbVersion > QShaderPrivate::QSB_VERSION_WITH_BINARY_JSON) {
494 qWarning(msg: "Can no longer load QShaderDescription from CBOR.");
495 d->desc = QShaderDescription();
496 } else {
497 qWarning(msg: "Can no longer load QShaderDescription from binary JSON.");
498 d->desc = QShaderDescription();
499 }
500 int count;
501 ds >> count;
502 for (int i = 0; i < count; ++i) {
503 QShaderKey k;
504 readShaderKey(ds: &ds, k: &k);
505 QShaderCode shader;
506 QByteArray s;
507 ds >> s;
508 shader.setShader(s);
509 ds >> s;
510 shader.setEntryPoint(s);
511 d->shaders[k] = shader;
512 }
513
514 if (d->qsbVersion > QShaderPrivate::QSB_VERSION_WITHOUT_BINDINGS) {
515 ds >> count;
516 for (int i = 0; i < count; ++i) {
517 QShaderKey k;
518 readShaderKey(ds: &ds, k: &k);
519 NativeResourceBindingMap map;
520 int mapSize;
521 ds >> mapSize;
522 for (int b = 0; b < mapSize; ++b) {
523 int binding;
524 ds >> binding;
525 int firstNativeBinding;
526 ds >> firstNativeBinding;
527 int secondNativeBinding;
528 ds >> secondNativeBinding;
529 map.insert(key: binding, value: { firstNativeBinding, secondNativeBinding });
530 }
531 d->bindings.insert(key: k, value: map);
532 }
533 }
534
535 if (d->qsbVersion > QShaderPrivate::QSB_VERSION_WITHOUT_SEPARATE_IMAGES_AND_SAMPLERS) {
536 ds >> count;
537 for (int i = 0; i < count; ++i) {
538 QShaderKey k;
539 readShaderKey(ds: &ds, k: &k);
540 SeparateToCombinedImageSamplerMappingList list;
541 int listSize;
542 ds >> listSize;
543 for (int b = 0; b < listSize; ++b) {
544 QByteArray combinedSamplerName;
545 ds >> combinedSamplerName;
546 int textureBinding;
547 ds >> textureBinding;
548 int samplerBinding;
549 ds >> samplerBinding;
550 list.append(t: { .combinedSamplerName: combinedSamplerName, .textureBinding: textureBinding, .samplerBinding: samplerBinding });
551 }
552 d->combinedImageMap.insert(key: k, value: list);
553 }
554 }
555
556 if (d->qsbVersion > QShaderPrivate::QSB_VERSION_WITHOUT_NATIVE_SHADER_INFO) {
557 ds >> count;
558 for (int i = 0; i < count; ++i) {
559 QShaderKey k;
560 readShaderKey(ds: &ds, k: &k);
561 int flags;
562 ds >> flags;
563 QMap<int, int> extraBufferBindings;
564 int mapSize;
565 ds >> mapSize;
566 for (int b = 0; b < mapSize; ++b) {
567 int k, v;
568 ds >> k;
569 ds >> v;
570 extraBufferBindings.insert(key: k, value: v);
571 }
572 d->nativeShaderInfoMap.insert(key: k, value: { .flags: flags, .extraBufferBindings: extraBufferBindings });
573 }
574 }
575
576 return bs;
577}

598/*!
599 Returns \c true if the two QShader objects \a lhs and \a rhs are equal,
600 meaning they are for the same stage with matching sets of shader source or
601 binary code.
602
603 \relates QShader
604 */
605bool operator==(const QShader &lhs, const QShader &rhs) noexcept
606{
607 if (!lhs.d || !rhs.d)
608 return lhs.d == rhs.d;
609
610 return lhs.d->stage == rhs.d->stage
611 && lhs.d->shaders == rhs.d->shaders
612 && lhs.d->bindings == rhs.d->bindings;
613}

625/*!
626 Returns the hash value for \a s, using \a seed to seed the calculation.
627
628 \relates QShader
629 */
630size_t qHash(const QShader &s, size_t seed) noexcept
631{
632 if (s.d) {
633 QtPrivate::QHashCombine hash;
634 seed = hash(seed, s.stage());
635 if (!s.d->shaders.isEmpty()) {
636 seed = hash(seed, s.d->shaders.firstKey());
637 seed = hash(seed, s.d->shaders.first());
638 }
639 }
640 return seed;
641}

774#ifndef QT_NO_DEBUG_STREAM
775QDebug operator<<(QDebug dbg, const QShader &bs)
776{
777 const QShaderPrivate *d = bs.d;

810/*!
811 \typedef QShader::NativeResourceBindingMap
812
813 Synonym for QMap<int, QPair<int, int>>.
814
815 The resource binding model QRhi assumes is based on SPIR-V. This means that
816 uniform buffers, storage buffers, combined image samplers, and storage
817 images share a common binding point space. The binding numbers in
818 QShaderDescription and QRhiShaderResourceBinding are expected to match the
819 \c binding layout qualifier in the Vulkan-compatible GLSL shader.
820
821 Graphics APIs other than Vulkan may use a resource binding model that is
822 not fully compatible with this. The generator of the shader code translated
823 from SPIR-V may choose not to take the SPIR-V binding qualifiers into
824 account, for various reasons. This is the case with the Metal backend of
825 SPIRV-Cross, for example. In addition, even when an automatic, implicit
826 translation is mostly possible (e.g. by using SPIR-V binding points as HLSL
827 resource register indices), assigning resource bindings without being
828 constrained by the SPIR-V binding points can lead to better results.
829
830 Therefore, a QShader may expose an additional map that describes what the
831 native binding point for a given SPIR-V binding is. The QRhi backends, for
832 which this is relevant, are expected to use this map automatically, as
833 appropriate. The value is a pair, because combined image samplers may map
834 to two native resources (a texture and a sampler) in some shading
835 languages. In that case the second value refers to the sampler.
836
837 \note The native binding may be -1, in case there is no active binding for
838 the resource in the shader. (for example, there is a uniform block
839 declared, but it is not used in the shader code) The map is always
840 complete, meaning there is an entry for all declared uniform blocks,
841 storage blocks, image objects, and combined samplers, but the value will be
842 -1 for those that are not actually referenced in the shader functions.
843*/
844
845/*!
846 \return the native binding map for \a key. The map is empty if no mapping
847 is available for \a key (for example, because the map is not applicable for
848 the API and shading language described by \a key).
849 */
850QShader::NativeResourceBindingMap QShader::nativeResourceBindingMap(const QShaderKey &key) const
851{
852 if (!d)
853 return {};
854
855 auto it = d->bindings.constFind(key);
856 if (it == d->bindings.cend())
857 return {};
858
859 return it.value();
860}
